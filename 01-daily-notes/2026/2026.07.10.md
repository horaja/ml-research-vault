## opener

Friday, July 10. GSPA unread — day 27. "When Attention Collapses" (STS) unread — day 16. Contrastive alignment spec unwritten — 7+ weeks.

Today's digest lead: **Compressor-VLA** (Gao et al.) instruction-guided visual token compression for robotic VLA manipulation — a direct instantiation of surf's [[claims/language helps]] without a structural prior. The paper identifies the same problem (task-agnostic pruning discards task-critical tokens) and proposes the same intervention (instruction conditioning drives token selection). The critical question: does Compressor-VLA's instruction-guided compression recover the structurally dense (contact-region, line-rich) patches that VLA-IAP (7/9) showed saliency misses? If yes, [[claims/language helps]] subsumes [[claims/structure helps]] and surf's differentiation argument collapses to: (a) structural priors work training-free where instruction attention cannot, or (b) structural density adds independent signal over instruction attention. If no, surf's structural prior selects a distinct token set that instruction alignment does not reach — the experimentally testable separation. Read this before writing the spec.

Second: **LEEVLA** (Lyu et al., arXiv today) identifies task-critical tokens via what changes in a learned latent environment representation across action steps — a dynamic alternative to surf's static structural prior. If latent-change-salient patches co-locate with structurally dense patches, LEEVLA implicitly discovers the same selection criterion. If they diverge, the two mechanisms operate at different timescales (static structural cue vs. dynamic scene evolution) and can be composed.

The three critical blockers — spec, STS, GSPA — are unchanged. Write the spec.

## questions

- Compressor-VLA vs. surf differentiation: Compressor-VLA uses instruction attention to compress visual tokens in VLA. Does this mechanism select the structurally dense patches (contact regions, line-rich geometry) that VLA-IAP showed saliency misses? If yes: [[claims/language helps]] subsumes [[claims/structure helps]] — surf's structural prior only matters training-free. If no: the structural prior selects a distinct, complementary token set that instruction attention doesn't reach — the testable core of surf's novelty argument. The paper's ablation (instruction-conditioned vs. task-agnostic) is the first piece of evidence; the cross-comparison with VLA-IAP's interaction-alignment regions is the second.
- LEEVLA latent evolution × surf static prior: LEEVLA's selection criterion is "what changes in latent representation between timesteps." Surf's is "what is structurally dense in the current frame." On LIBERO manipulation scenes with slow background motion and contact-point state change: do the two criteria select the same patches? If the scene is nearly static except at the contact point, latent evolution isolates the same structurally critical region surf's prior selects — and LEEVLA may be an implicit structural-density detector via temporal subtraction.
- Write-Protected bottleneck × Compressor-VLA: Fang's write-protection argument applies to discrete world model bottlenecks, not attention-based instruction conditioning. Does the structural limitation extend to soft attention injection (as in Compressor-VLA)? If soft attention injection also corrupts physical symbol representations, surf's external contrastive scoring avoids the failure mode entirely. If not, attention injection is safe and the architectural argument for contrastive scoring weakens to a performance argument.
- Lu et al. TNN × surf V4 claim: TNNs produce topographic organization as an emergent property of spatial-smoothness + self-supervised training, and explain human gaze behavior. If a TNN trained on manipulation scenes develops topographic structure, does that structure co-locate with surf's line-density prior? The convergence would make [[claims/v4 relevance]] a corollary of a task-training argument rather than a biological analogy — stronger grounding for the spec.
- Eye-hand anticipatory saccade × surf patch selection: Zhao et al.'s anticipatory saccade arrives at the target region *before* the hand. If surf's structural prior selects that target region from the observation, it's functioning as an anticipatory gaze model — selecting for information the motor system will need before it needs it. Does DAA/PAGE/DeepGaze3.5-VL capture this temporal anticipation aspect, or does it only evaluate spatial co-location at the moment of contact?
