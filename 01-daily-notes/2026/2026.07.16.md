# 2026-07-16

## opener

Thursday, July 16. Contrastive alignment spec unwritten — 11 weeks. GSPA unread — day 33. "When Attention Collapses" (STS) unread — day 22. Visual Access Boundaries unread — day 2.

Today's digest tightens the cluster of papers challenging surf's contrastive design. **Screening Is Effective for Visual Recognition** (Shimomura & Hotta, arXiv) proposes evaluating ViT patches independently for binary relevance — no softmax normalization — and screening them before attention. This is mechanistically the same move as surf's line-density prior: both independently score patches before the attention stack. The question is whether the learned screening criterion and the structural prior select the same patches. If they do, the prior is a hand-engineered approximation of what the learner finds anyway. If they don't, surf's structural prior carries information the screener misses.

**Attention-Free and Lightweight Token Reduction** (Hao et al., arXiv) replaces attention-map-based scoring with a lightweight feature criterion — directly competing with surf's contrastive scorer. And yesterday's VisCo (day 2) and What Makes a Prior Work (day 2) hit the same nerve. Three papers in two days all arguing that simpler, lighter mechanisms may achieve what surf's contrastive training objective is designed to do. The spec must now state explicitly what contrastive alignment adds over (a) learned independent screening, (b) lightweight feature matching, and (c) intrinsic LLM encoding. Write the spec.

- [Attention-Free and Lightweight Token Reduction for Efficient Vision-Language Models](https://arxiv.org/abs/2607.13500v1) — X. Hao, Z. Zhang, Z. Wang · arXiv (2026-07-15)
  Token reduction via a lightweight feature-based relevance criterion that bypasses attention-map computation. Competing approach to surf's contrastive scorer: if feature matching rivals cross-modal contrastive scoring on token selection quality, surf's training cost for the contrastive objective is unjustified. Spec must show contrastive action-discriminative scoring selects different and contact-geometry-relevant tokens that AFTR's feature criterion misses on LIBERO.

- [Do All Visual Tokens Matter Equally? Object-Evidence Preserving Token Merging for Vision-Language Retrieval](https://www.semanticscholar.org/paper/ede30f0714cdb41842d2efab10ac4b5104442f2b) — S2 (2026-07-06)
  Establishes empirically that visual tokens are not equally important for task performance; proposes an object-evidence criterion (tokens bearing discriminative object features) for token merging in VL retrieval. Different domain, same core claim as surf. Key question: do object-evidence tokens co-locate with action-evidence tokens (contact geometry) on manipulation scenes? If yes, object-evidence merging is a pretrained baseline surf must beat. If no, surf's action-discriminative criterion selects a distinct patch set with independent value.

**adjacent**
- [S-squared-VLA: Decoupling Semantic and Spatial Streams in Vision-Language-Action Models for Autonomous Driving](https://arxiv.org/abs/2607.13926v1) — J. Yu, R. Wang, D. Chu · arXiv (2026-07-15)
  Separates semantic stream (high-level scene understanding) from spatial stream (low-level action prediction) in an autonomous driving VLA. Domain is driving not manipulation, but the semantic/spatial decoupling is the right vocabulary for surf's prior/action module boundary: prior module operates on the spatial stream (which patches to select), action module on the semantic stream (what to do given selected patches). Read to borrow the framing for the spec architecture section.

- [Subtoken Vision Transformer for Fine-grained Recognition](https://www.semanticscholar.org/paper/48715793384880f6fc246bc9c13e5875ca21906a) — S2 (2026-07-10)
  SubViT: standard ViT compresses each 16×16 patch into one token regardless of local content diversity; SubViT splits high-diversity patches into multiple sub-tokens adaptively. Relevant to surf's granularity question: if contact regions in manipulation (gripper fingertip, edge grasp) are smaller than 16px, surf's patch-level line-density selection picks the right region at the wrong resolution. SubViT raises whether surf needs adaptive sub-patch granularity at selected contact patches.

**horizon**

- [Kepler-Encoder-v0.1: Towards a Multimodal Embedding Model for Robots](https://arxiv.org/abs/2607.13522v1) — I. S. Singh et al. · arXiv (2026-07-15)
  Robot-first multimodal encoder integrating visual, force, and contact modalities; reports that contact/force is essentially not recoverable from raw vision features alone (R² ≤ 0.10 across all tested robots). Establishes a visual-only ceiling for surf: if contact dynamics that determine manipulation success are invisible in visual tokens, surf's structural prior cannot select "action-relevant" patches in the contact-predictive sense. Reframes what surf's visual selector can actually optimize for: object geometry and affordance structure, not contact dynamics.

## questions

- Screening × surf structural prior: if a learned screening function achieves the same patch selection as line density without supervision on structural features, is the structural prior useful beyond reducing training cost? Experiment: train learned screener from scratch vs. initialize with line-density scores; does the learner converge to a structurally similar criterion?
- Attention-free reduction × [[claims/language helps]]: if lightweight feature-based matching (no cross-modal attention) achieves comparable token selection, surf's contrastive training objective is over-engineered. Diagnostic: compare AFTR-selected tokens vs. contrastive-scorer-selected tokens on LIBERO; do they select the same contact-geometry patches?
- Object-evidence × action-evidence: tokens that carry discriminative object features (object-evidence criterion) may or may not be the same as tokens carrying action-relevant contact geometry (surf's claim). Comparison on LIBERO is the empirical test. If they differ, surf's action-evidence criterion has independent value.
- SubViT × surf granularity: surf operates at 16×16 patch granularity. For manipulation contact regions (gripper fingertip, edge grasp sites), is 16px the right granularity, or does sub-patch selection at chosen contact patches improve action-discriminative precision?
- Kepler-Encoder × surf scope: if force/contact is not recoverable from visual tokens (R² ≤ 0.10), surf's visual structural prior cannot select tokens that are contact-predictive. What can visual tokens predict about actions? Object geometry and affordance structure. Is that sufficient for surf's contribution, or does the claim need to be reframed?
