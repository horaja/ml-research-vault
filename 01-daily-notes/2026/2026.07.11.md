Yesterday's digest lead (Compressor-VLA) is now day 2 critical: it tests [[claims/language helps]] without [[claims/structure helps]]. Whether instruction-guided compression recovers the structurally critical patches VLA-IAP identified is the most important empirical question in the spec's novelty section. Read it before writing anything today.

Today's digest surfaces **ThinkProprio** (Wang et al., Feb 2026): the first VLA method to use proprioceptive state — discretized into VLM vocabulary tokens — to gate visual patches *before* VLM computation. This opens a third conditioning axis adjacent to surf's space: instruction + proprioception → patch selection. The direct question: does proprioceptive grounding recover the contact-region, structurally dense patches that Compressor-VLA's instruction-only conditioning misses? If proprioception and structure co-select the same patches (both capturing affordance/contact relevance), ThinkProprio validates that *something beyond task-agnostic saliency is needed* while specifying what surf must be differentiated from: instruction conditioning, proprioceptive conditioning, and structural density are now three distinct candidate signals requiring a controlled ablation.

Second: **Latent Memory Palace** (Zhu et al., arXiv 2607.08724) reframes robot control deliberation as autoregressive variational inference over a latent memory space. The framing maps onto [[claims/iterative selection]]: some steps require immediate reactive control; others require deliberate reasoning from accumulated visual evidence. If surf's iterative reselection implements the deliberate branch, the correct memory substrate is latent — and the allocation signal is uncertainty, not a structural prior.

**adjacent**

  Replaces full-image visual states during interleaved reasoning with delta-image tokens (only what changed between reasoning steps). The sparse-update principle transfers to VLA: if only scene-change regions carry decision-relevant information across action steps, surf's iterative reselection should concentrate on delta patches rather than rediscovering the full static structure. Operationally: VLA-IAP's interaction alignment = regions where contact-induced change occurs = the delta-image region. Structural density prior may be an efficient pre-contact proxy for the same region.

- [Beyond Attention or Similarity: Maximizing Conditional Diversity for Token Pruning in MLLMs](https://www.semanticscholar.org/paper/3b23dd0634e7de523598df0a907a5bf558eb9c88) — Q. Zhang et al. · arXiv (72 cites)

## questions

- ThinkProprio conditioning × surf structural prior: ThinkProprio gates patches using instruction + proprioception; surf uses instruction + structural density (line prior). Do both criteria select the same contact-region, affordance-critical patches on manipulation scenes? If yes: proprioception is a robot-state-derived proxy for structural density, and surf's static prior is the training-free version of what ThinkProprio learns. If no: the two criteria are independent — the ablation table for [[experiments/contrastive alignment]] must include instruction-only (Compressor-VLA), proprioception-only (ThinkProprio), structure-only (surf), and composites.
- Latent Memory Palace × iterative selection substrate: Latent Memory Palace implements deliberate reasoning as latent-space variational inference, not language-space. If surf's iterative patch reselection is the observation-level analogue of deliberation, what is the correct memory substrate? A latent VLA representation (what the model currently believes about the scene) or a structural prior (what the scene physically contains)? The two make different predictions: latent-memory reselection learns to concentrate on action-relevant dynamics; structural-prior reselection concentrates on geometric invariants.
- Conditional diversity (Beyond Attention) × structural density: conditional diversity maximization is already a competitive token pruning criterion (72 cites). Does surf's structural density prior add independent information beyond what conditional diversity selects? If both recover structurally critical manipulation patches, the diversity criterion is a stronger baseline than attention-based pruning — and surf's structural prior needs to be compared against it, not just against attention-based methods.
- DeltaV delta-image principle × surf iterative reselection: DeltaV's delta-image tokens are exactly the regions that changed between reasoning steps. In VLA manipulation, those are the contact/interaction regions that VLA-IAP also preserves. If surf's structural prior pre-selects those regions *before* contact occurs (because structurally dense patches include affordance edges and contact geometry), surf is an anticipatory delta-image selector. The test: does surf's line-density prior on a pre-contact observation frame co-locate with the actual delta regions in the post-contact frame?
