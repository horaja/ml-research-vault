## papers

### digest (auto)

**core**
- [FlashVLM: Text-Guided Visual Token Selection for Large Multimodal Models](https://www.semanticscholar.org/paper/4fddca70a71254c39d619c4f53727b00b4fea9fe) — Cai et al. · arXiv
  Dynamically selects visual tokens based on shallow text-token cross-attention rather than deep attention maps; avoids the instability under aggressive pruning that attention-map methods show. Directly engages [[claims/language helps]]: a training-free, language-conditioned selector that works at high compression ratios is a strong baseline the zero-language ablation must be compared against, not just a zero-language version of the same architecture.

- [D2Pruner: Debiased Importance and Structural Diversity for MLLM Token Pruning](https://www.semanticscholar.org/paper/c0cbf1e33a7366eeca082379637bb8ad7c74cd12) — Zhang et al. · AAAI
  AAAI paper: importance-based pruning fails fine-grained localization due to positional bias; diversity-based pruning fails it due to structural blindness; combining debiased importance + structural diversity fixes both. Extends the PosPrune thread from 5/27 into localization tasks — the [[experiments/e001]] center-bias baseline may need a structural-diversity component alongside the ARP-style positional correction to be a rigorous control.

- [OneTwoVLA: A Unified Vision-Language-Action Model with Adaptive Reasoning](https://www.semanticscholar.org/paper/030f9cc8b8d6d46d6239ef6ae29d9e20867194de) — Lin et al. · arXiv
  Single VLA model that switches between a fast acting mode (System 1) and a slow reasoning mode (System 2) based on task demands; unified rather than two-stage. A direct candidate architecture framing for surf's efficiency/RL module: "when to request more visual evidence" could be formulated as a binary mode switch rather than a continuous budget allocation, which is simpler and may generalize better.

**adjacent**
- [HoliTom: Holistic Token Merging for Fast Video Large Language Models](https://www.semanticscholar.org/paper/6f172c00ac227d7f71157916e55d2330180fe46b) — Shao et al. · arXiv
  Jointly merges spatial (intra-frame) and temporal (inter-frame) tokens using a holistic redundancy criterion; outperforms spatial-only or temporal-only approaches. Directly challenges the implicit scope assumption in [[claims/iterative selection]]: if holistic spatial+temporal merging beats per-frame spatial selection, surf's per-frame structural prior is solving the wrong granularity for VLA tasks.

- [DreamVLA: A Vision-Language-Action Model Dreamed with Comprehensive World Knowledge](https://www.semanticscholar.org/paper/f374c4d9407566e349ab861c5d51720b211e37f9) — Zhang et al. · arXiv
  Integrates dynamic, spatial, and semantic world knowledge into a VLA via latent prediction rather than image generation, improving generalization. Sets a capability baseline for what a VLA can know without token pruning; if world-knowledge integration requires dense visual context, surf's compute-efficiency argument needs to specify which world-knowledge signals survive its token budget.

- [Action-Sketcher: From Reasoning to Action via Visual Sketches for Long-Horizon Robotic Manipulation](https://www.semanticscholar.org/paper/ce9ba7954c4604a4ec6aa213b2bab4fee2df1614) — Tan et al. · arXiv
  Uses explicit spatial sketches (visual annotations) to bridge language reasoning and robot action in cluttered long-horizon tasks; the sketch is a form of structured spatial prior over the workspace. An explicit-sketch prior is a strong competitor to surf's implicit structural prior: if drawing the relevant structure beats detecting it, the implicit vs. explicit axis becomes the key design question for [[claims/structure helps]].

**horizon**
- [Chain-of-Visual-Thought: Teaching VLMs to See and Think Better with Continuous Visual Tokens](https://www.semanticscholar.org/paper/f6955d3a63b5ed7116b9d66a147090ca2acde0aa) — Qin et al. · arXiv
  VLMs generate ~20 continuous latent visual tokens during reasoning (not image generation) to capture dense spatial cues before answering. Bidirectional compute framing: surf reduces input tokens (visual budget in), COVT adds reasoning tokens (visual budget out). If dense spatial reasoning requires generated visual tokens at decode time, adaptive compute may need to account for both axes — not just efficient perception, but efficient spatial reasoning.

- [MergeVQ: A Unified Framework for Visual Generation and Representation with Disentangled Token Merging and Quantization](https://www.semanticscholar.org/paper/c141b15c26a97ee131105b6152b47bac4ee8e1fa) — Li et al. · CVPR
  Disentangles token merging (which tokens carry generation-critical detail) from quantization (which tokens carry representation-critical semantics) in a unified MIM + generation framework. Relevant to [[eval/reconstruction]]: the merging-before-quantization design empirically separates "important for reconstruction fidelity" from "important for semantic recognition" — the same confound surf's reconstruction-residual metric needs to avoid.

## questions
- FlashVLM selects tokens via text-token cross-attention in shallow layers: does this correlation score capture instruction-specificity (which object/region the instruction names), or does it capture general image-text salience? If the latter, it may not beat a zero-language selector on manipulation tasks where the instruction names an action, not a salient object.
- D2Pruner combines debiased importance + structural diversity (AAAI) and fixes fine-grained localization failures; does the "structural diversity" component correspond to line/boundary coverage, or is it semantic diversity (object regions vs. background)? If line structure is a diversity prior, surf's [[claims/structure helps]] is a special case of D2Pruner's diversity term.
- OneTwoVLA's System 2 mode is triggered adaptively: on what signal does the mode switch happen — uncertainty, task complexity, instruction specificity? If it's instruction-conditioned, it overlaps with surf's language-conditioned selection claim.
- HoliTom holistic merging jointly considers spatial and temporal redundancy: in VLA tasks with near-static camera views, is temporal redundancy the dominant axis? If most tokens are temporally redundant, spatial selection within a frame may be a small optimization.
