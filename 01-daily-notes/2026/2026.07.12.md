## questions

- ResPrune × surf [[claims/language helps]]: does text-conditioned subspace reconstruction select the same task-relevant tokens as contrastive training? If subspace fidelity ≈ contrastive relevance, training-free text conditioning is sufficient and the contrastive objective needs a stronger justification. If not (subspace reconstruction optimizes input fidelity; contrastive scoring optimizes action discriminability), the gap is surf's value-add — but requires action-relevant token correctness as the isolating metric.
- Focus-Scan-Refine × surf iterative selection: does FSR's coarse-to-fine decomposition subsume surf's iterative patch reselection? If surf's first pass = focus (structural density) and second pass = refine (language conditioning), surf should explicitly formalize this structure in the spec. If surf's passes operate on a different axis (not attention anchors), FSR is a competing design and belongs in the ablation table.
- S²-VLA × surf iterative selection: if cumulative error in long-horizon VLA stems from static feature fusion rather than token selection noise, surf's iterative reselection may be fixing the wrong bottleneck. The test: iterative reselection + static fusion vs. one-shot selection + state-space dynamics on the same long-horizon manipulation tasks. Which ablation actually recovers performance?
- Z-1 RL objective × surf budget allocation: is Z-1's efficiency reward at the task level or token level? If task-level only, surf needs a separate token-selection quality signal to train the budget policy — the reward attribution problem is not solved by Z-1's approach alone.
