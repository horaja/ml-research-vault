Today's digest surfaces two papers that belong in the spec's ablation table before anything else: **VLA-IAP** (Cheng et al.) introduces interaction-alignment-based training-free pruning that uses cross-layer attention interaction patterns — not language, not structure — to score token importance in VLA. If interaction alignment outperforms structural scoring on LIBERO contact tasks, surf's structural prior is adding nothing a purely attention-derived criterion doesn't already capture. Read before writing the ablation table. **History-Conditioned ST Pruning** (Wang et al.) is the training-free temporal competitor to [[claims/iterative selection]] that's been on the risk list since day 5: it prunes redundant visual tokens by comparing current observation against historical frames, without any learned scorer or explicit prior. If it matches surf's iterative reselection on VLN tasks at lower compute, the case for a contrastive structural scorer weakens considerably.

**Gaze on the Prize** (Lee et al.) is the adjacent paper worth attention: it trains a contrastive scorer using the RL return signal to identify task-relevant visual regions. Surf's contrastive alignment uses instruction-language supervision; this paper uses reward supervision. The comparison raises a concrete ablation: does instruction conditioning in surf's scorer provide independent signal over a return-guided contrastive baseline trained without language? If return-supervised gaze outperforms instruction-supervised selection, the language conditioning claim needs more careful isolation.

## questions

- VLA-IAP × [[claims/structure helps]]: VLA-IAP scores tokens by cross-layer interaction alignment — how attention patterns interact across layers — without structural or language signal. Do interaction-aligned tokens and surf's line-dense patches co-locate on LIBERO contact regions? If they co-locate, interaction alignment and structural prior select the same tokens via different signals (structure may be approximating what cross-attention already expresses). If they diverge, the structural prior selects tokens that interaction alignment misses — the differentiating case surf needs.
- History-Conditioned ST × [[claims/iterative selection]]: History-conditioned pruning retains tokens where visual content diverges from prior observations — effectively selecting "what's new." Surf's iterative selection scores tokens by instruction-contact-geometry relevance — effectively selecting "what matters." On a sequential LIBERO contact task, do "what's new" and "what matters" select the same regions? If they overlap, history-based pruning can substitute for surf's scorer. If they diverge, the cases are: contact-geometry is always new (history overlaps), or contact-geometry is present from the start but only becomes relevant (history misses it).
- Gaze on the Prize × [[experiments/contrastive alignment]]: This paper trains contrastive scoring from return supervision, not instruction supervision. Can surf use return-guided contrastive pretraining as a base and add instruction conditioning as a refinement? Or are the two contrastive objectives incompatible (return signal is episodic, instruction signal is per-step)? The paper's ablation table may show whether contrastive foveation from return generalizes to contact tasks.
- RoboTTT × [[claims/iterative selection]]: If 8K-step context is achievable without latency cost (via selective attention over long context), is surf's per-step iterative reselection addressing the same problem? The efficiency arguments differ: surf reduces tokens per step; RoboTTT reduces latency per step via cached context. Both target the same bottleneck (VLA inference cost on time-critical robotics). Does surf's adaptive visual budget reduce context length needs, or are they orthogonal?
- LA4VLA × [[claims/language helps]]: LA4VLA shows language-action pretraining captures most task structure. If surf's contrastive scorer uses a language encoder pretrained on language-action pairs (e.g., OpenVLA backbone), does the instruction conditioning in the contrastive objective actually add over the backbone's language-action alignment? The ablation is: contrastive scorer with language vs. contrastive scorer with language replaced by the pretrained backbone's action embedding.
- Topographic networks × [[claims/v4 relevance]]: ETTN shows topographic organization emerges from a top-down task objective. If the ETTN's topographic map captures similar shape/texture organization to V4 (as the paper claims), can surf use an ETTN-derived dispersity map as a computational prior — without needing the biological V4 literature at all? This would decouple the V4 motivation from the biological grounding requirement and make the prior fully computational.
