## opener

Thursday, July 9. GSPA unread — day 26. "When Attention Collapses" (STS) unread — day 15. Contrastive alignment spec unwritten — 7+ weeks.

Today's digest lead: **VLA-IAP** (Cheng et al., arXiv 2603.22991) provides independent empirical support for surf's structural claim from a competing direction. The paper identifies that existing saliency-based VLA token pruning methods consistently prune "visually sparse yet structurally critical regions" — the exact same patch class surf's line-drawing prior is designed to preserve. VLA-IAP proposes "interaction alignment" (proximity to predicted contact/interaction zones) as the correct selection criterion. Surf's line-density prior is a proposed efficient proxy for the same region class derivable from RGB without contact prediction. The critical question: how much do the two criteria diverge on LIBERO manipulation scenes? Co-location would validate line drawings as an efficient proxy for interaction alignment; divergence would identify the gap that surf's structural scorer must close.

Second: **AnchorPrune** (Oh & Kim, arXiv today) offers a principled solution to the relevance/diversity tradeoff in VLM token pruning by building prototype-cluster anchors and expanding context around each. The anchor selection step is currently query-driven; surf's structural prior (line density, contact-region-biased) is a candidate anchor criterion for VLA-specific adaptation — a concrete architectural pattern to borrow for the [[experiments/contrastive alignment]] spec.

The spec and the unread blockers remain unchanged. Write the spec.

## questions

- VLA-IAP interaction alignment × surf structural prior co-location: VLA-IAP preserves tokens near predicted contact/interaction zones; surf selects structurally dense (line-rich) patches. On LIBERO manipulation scenes, do these two selection criteria select the same patch sets? If yes: line drawings are an efficient RGB-only proxy for interaction alignment, no contact prediction needed. If no: quantify the divergence — which set captures contact-relevant affordance edges that the other misses? The divergence is the experimental claim for [[claims/structure helps]].
- VLA-IAP × surf differentiation: VLA-IAP is training-free; surf's contrastive scorer requires training. If VLA-IAP interaction alignment already captures structurally critical regions, what does surf add? The answer must be: (a) surf uses language conditioning that VLA-IAP ignores, or (b) surf's structural prior is more generalizable across task types than contact-zone prediction, or (c) surf composes with VLA-IAP as a complementary signal. Which is the strongest differentiation argument for the spec?
- AnchorPrune anchoring × surf line prior: AnchorPrune's anchor step selects prototype representatives via query relevance. If surf replaces this with line-density-based anchoring (structurally dense patches as anchors), does AnchorPrune's context expansion still improve over anchor-only selection? This is a concrete experiment: line-density anchors + AnchorPrune expansion vs. line-density pruning alone.
- V4 natural-image tuning × dispersity metric: Willeke et al. find V4 topological organization holds for natural-image tuning, not just parametric stimuli. Does the dispersity metric from @jiangComputationalConstraintsUnderlying generalize to natural-image DNN activations? If yes, computing per-patch dispersity on manipulation scene images is a concrete prior derivation pipeline from V4 functional maps.
- WorldBagel world-model prediction × iterative patch selection: if a world model predicts "which regions will change after this action," does this prediction co-locate with surf's current structural prior selections? If yes, the structural prior is an implicit world-model proxy. If no, which is more informative for task success?
