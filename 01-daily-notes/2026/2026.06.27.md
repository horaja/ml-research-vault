## papers

### digest (auto)

**core**
- [The Better You Learn, The Smarter You Prune: Towards Efficient VLA Models via Differentiable Token Pruning](https://www.semanticscholar.org/paper/6163650edbfba0a837b0722c9c49ed49040ba70d) — T. Jiang, X. Jiang, Y. Ma et al. · arXiv
  LightVLA introduces a differentiable token-pruning objective for VLA, learning to prune at training time rather than via a fixed structural prior at inference. This is surf's selection claim — which tokens matter in VLA observation — implemented as a fully differentiable baseline. The critical comparison: does learned differentiable pruning converge on the same spatial regions as surf's geometric prior? If yes, the prior is an initialization; if no, the prior encodes signal that gradient descent alone misses. Must appear in spec's differentiation table alongside TOPS.

- [FlashVLM: Text-Guided Visual Token Selection for Large Multimodal Models](https://www.semanticscholar.org/paper/4fddca70a71254c39d619c4f53727b00b4fea9fe) — K. Cai, J. Zhang, J. Yang · arXiv
  Uses textual query to select visual tokens while explicitly rejecting deep-attention-map methods for their instability under dynamic inputs. Directly supports [[claims/language helps]]: text-guided selection, without the geometric prior, may already achieve task-conditioned spatial selection. Key question: does FlashVLM's text-guided scoring use shallow features (Q/K similarity) or a trained compatibility function? If shallow, surf's contrastive scorer is a stronger language-conditioning baseline. If trained, FlashVLM is a direct competitor to surf's scorer without the structural prior stage.

- [Non-Markovian Long-Horizon Robot Manipulation via Keyframe Chaining](https://www.semanticscholar.org/paper/e24f15946a4a18c63b3c7215dbc5b0c38bbe07ee) — Y. Chen, W. Tan, L. Zhu · arXiv
  VLA models fail at long-horizon tasks because they rely only on immediate observations, ignoring Non-Markovian task dependencies; addresses this by chaining keyframes as persistent memory. Relevant to [[claims/iterative selection]]: keyframe chaining is iterative selection at the temporal axis (which past frames to retain) rather than the spatial axis (which patches to attend to). The two axes are compositional — if keyframe chaining solves when to carry visual context forward, surf's patch selection solves what within each frame — but the paper may also provide a "when to refine" criterion based on task-progress discontinuity.

**adjacent**
- [STARRY: Spatial-Temporal Action-Centric World Modeling for Robotic Manipulation](https://www.semanticscholar.org/paper/f134eaaeb2cf6d4ac3526993199033b7f858a35b) — Y. Tian, Y. Jin, B. Yu · arXiv
  Couples spatial-temporal world-model predictions (future geometric state) with VLA action execution; the world model's geometric predictions guide which spatial tokens are attended to during planning. Adjacent to surf's structural prior: if a world model predicts future geometry accurately, it provides a learned future-state prior for token selection — which is stronger than surf's static line-drawing prior but requires accurate world prediction. Frames the structural prior as a lightweight proxy for a full geometric world model.

- [ProgVLA: Progress-Aware Robot Manipulation Skill Learning](https://www.semanticscholar.org/paper/ae4fbe6ef22c468a58c01d088228940a5571befb) — S. Kim, J. Choi, S. Baek · arXiv
  Compact VLA with explicit task-progress representation, designed for tight compute and memory budgets. Represents an alternative efficiency axis to surf: progress-awareness achieves compute reduction by attending differently at different task stages (coarse early, precise late), without an explicit spatial-selection mechanism. The compositional question: does surf's within-frame spatial selection add non-redundant signal over ProgVLA's stage-aware temporal attention?

- [D2Pruner: Debiased Importance and Structural Diversity for MLLM Token Pruning](https://www.semanticscholar.org/paper/c0cbf1e33a7366eeca082379637bb8ad7c74cd12) — E. Zhang, F. Yu, A. Wu · AAAI
  Diagnoses two failure modes in token pruning for MLLMs: importance bias (high-attention tokens cluster redundantly) and structural diversity deficit (pruned sets lack spatial spread). The fix: debiased importance + structural diversity constraint. AAAI accepted. "Structural diversity" is surf's structural prior restated as a diversity objective: ensure the retained token set covers structural variety. If diversity-driven pruning implicitly achieves what surf's geometric prior explicitly enforces, the prior is redundant as a diversity mechanism — but may still be necessary as a task-relevance signal.

**horizon**
- [DnA: Denoising Attention for Visual Tasks](https://arxiv.org/abs/2606.27372v1) — R. Campos, S. Maity, X. Li · arXiv
  Replaces softmax attention with a denoising variant that suppresses low-signal attention weights, producing cleaner spatial focus without structured pruning. Horizon connection: if noisy softmax attention is the mechanism behind spatial diffusion in VLAs (the failure mode surf's structural prior corrects for), then denoising attention is an architectural fix at the same level. The two approaches make different assumptions about where the problem sits: surf assumes the visual input contains signal that attention fails to surface; DnA assumes attention itself is the noise source. If DnA's denoised attention converges on structurally dense regions without any geometric prior, the prior is unnecessary — and vice versa.

- [E-TTS: Embodied Test-Time Scaling for Robotic Manipulation](https://arxiv.org/abs/2606.27268v1) — W. Ye, P. Li, T. Yuan · arXiv
  Studies test-time scaling for embodied tasks: how much does allocating more inference compute (more reasoning steps, larger context) improve manipulation success? Horizon framing: test-time compute scaling is a parallel axis to surf's visual-token budget — both concern how to allocate limited compute at inference time. Surf's budget is spatial (which tokens enter the model); test-time scaling's budget is temporal (how many reasoning steps). If test-time scaling is the dominant efficiency lever, it competes with the narrative that spatial selection is where inference compute should be spent.

## questions

- LightVLA convergence regions: do LightVLA's differentiable-pruning learned masks concentrate on the same spatial regions as surf's line-drawing prior on manipulation scenes? If yes, the prior is a good initialization for the differentiable objective. If no, they select different signal axes — and the differentiable approach may be missing geometric structure.
- FlashVLM scoring mechanism: is FlashVLM's text-visual compatibility function trained (contrastive or supervised) or a zero-cost similarity metric (Q/K dot product)? If trained, it is a direct competitor to surf's scorer without the geometric prior stage. If zero-cost, surf's trained contrastive scorer is strictly stronger on language conditioning.
- D2Pruner diversity constraint: does D2Pruner's structural diversity constraint produce spatially diverse token sets that happen to cover geometrically dense regions, or does it distribute tokens regardless of geometric content? If diversity ≈ geometric coverage, D2Pruner operationalizes surf's structural prior as a diversity objective.
- Non-Markovian keyframe chaining × surf's iterative selection: are keyframe chaining (which past frames matter) and surf's iterative spatial selection (which patches within the current frame matter) truly compositional, or does chaining already implicitly select spatially relevant patches via the keyframe content?
