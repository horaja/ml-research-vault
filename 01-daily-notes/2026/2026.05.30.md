## papers

### digest (auto)

**core**
- [The Better You Learn, The Smarter You Prune: Towards Efficient Vision-language-action Models (LightVLA)](https://www.semanticscholar.org/paper/6163650edbfba0a837b0722c9c49ed49040ba70d) — Jiang, Jiang et al. · arXiv
  Differentiable token pruning for VLA that ties pruning aggressiveness to learning quality: better-learned tokens survive higher compression. Directly relevant to the [[questions]] circular-dependency concern — if pruning adapts to training signal rather than to a separate selector, the dependency between selector inputs and downstream features dissolves. Different mechanism from yesterday's Beyond Surrogate Gradients (which replaced Gumbel-Softmax generally); this one is VLA-specific and quality-adaptive.

- [DynaFLIP: Rethinking Robotics Perception via Tri-Modal-Dynamics Guided Representation](https://arxiv.org/abs/2605.30350v1) — Lee, Lee, Huang, Huang et al. · arXiv 2026
  Argues that visual encoders pretrained for static recognition or VL alignment miss motion-relevant features needed for manipulation; proposes dynamics-aware multimodal pretraining (RGB + depth + proprioception) to push action-relevant representation upstream into the encoder. The core question it answers — "what makes a visual feature relevant for action?" — is the same question underlying [[claims/structure helps]] and the e001 baseline design. If dynamics pretraining is required for action relevance, a line-drawing encoder may capture shape structure but miss the temporal/motion axis entirely.

- [CogVLA: Cognition-Aligned VLA via Instruction-Driven Routing & Sparsification](https://www.semanticscholar.org/paper/e30e7b2e70556078190f63ac50f613a2f1efd6b3) — Li, Zhang et al. · arXiv
  Implements instruction-driven routing and token sparsification jointly in a VLA: uses language input to route attention and prune tokens at once, without extensive post-training. This is the closest existing implementation of [[claims/language helps]] — an instruction-conditioned selector that sparsifies visual tokens at inference time. Baseline threat: if instruction-driven routing without explicit contrastive training already achieves competitive results, surf's contrastive alignment objective needs to show what the explicit alignment signal adds over implicit routing from the LLM.

**adjacent**
- [SpecPrune-VLA: Action-Aware Self-Speculative Pruning for VLA Acceleration](https://www.semanticscholar.org/paper/e5e4c5f50fef44a34d1235659e2cdb6cf3db5a6b) — Wang, Xu et al. · arXiv
  Addresses local-context failure in VLA pruning (per-step pruning loses >20% success rate) by using global context across action steps to guide pruning decisions. Relevant to surf's temporal scope question: if per-step structural prior selection needs global action history to avoid catastrophic pruning errors, the prior module cannot be fully decoupled from the action execution trajectory.

- [PARCEL: Pool-Anchored Resampling with Conditioned Elastic Queries](https://arxiv.org/abs/2605.30126v1) — Kuzucu, Tonioni, Schiele, Tombari, Naeem · arXiv 2026
  Elastic visual-token compression that trains a single model across a range of token budgets; pools both spatial and anchor tokens to prevent spatial-only compression from acting as an implicit downsampler. Shows that "elastic" budget training is tractable without quality collapse at aggressive compression ratios. Relevant to surf's visual budget design: if a model can be trained once and queried at any compression ratio, the efficiency/RL module's budget allocation problem is decoupled from retraining.

- [Beyond Sight: Finetuning Generalist Robot Policies with Heterogeneous Sensors via Language Grounding](https://www.semanticscholar.org/paper/3267c7120cb67167d0fd3d65faac0ac6e5a24abd) — Jones, Mees, Sferrazza, Stachowicz, Abbeel, Levine · ICRA
  Finetunes robot policies to use touch and audio alongside vision by grounding heterogeneous sensor signals through language; when vision is occluded, the policy shifts to touch/sound. From tracked authors Mees + Levine. The language-grounding mechanism here is a close cousin to surf's language-conditioned visual selection — language identifies which modality (or which region) matters for the current task. If language can ground cross-modal sensor selection, it should ground within-modality patch selection too.

**horizon**
- [Optimizing Anytime Reasoning via Budget Relative Policy Optimization](https://www.semanticscholar.org/paper/9102b58c727577172e6c0cf53c9b928b752a33aa) — Qi, Liu et al. · arXiv
  RL training for anytime reasoning under compute budgets: reward is relative to budget (not just final accuracy), so the model learns to improve at every intermediate budget level. The "anytime" framing and budget-relative reward maps directly to surf's efficiency/RL module design — the stopping criterion should reward good decisions at every compute level, not just when the full budget is spent. Budget Relative Policy Optimization (BRPO) is a concrete algorithm worth examining before committing to a formulation.

- [Exploratory Experience Shapes the Geometry of Predictive Representations](https://www.semanticscholar.org/paper/32fbfdc2a3af37f998167a40607656c9af6be023) — Shilova, Sharafeldin, Balakrishnan, Choi · arXiv
  Active sensing theory paper: exploration strategy shapes the geometry of internal predictive representations; the predictive coding framework couples action selection with representation learning through an action-perception loop. From Hannah Choi's group (computational neuroscience). Theoretical grounding for the question Dr. Lee raised (2026-05-24 meeting): "where-to-look conditioned on task/question." If geometry of predictive representations is shaped by exploration policy, surf's structural prior is not just a spatial filter — it is a choice of exploration strategy that will shape what the encoder learns.

## questions
- CogVLA routes tokens using instruction input — is the routing signal from the instruction embedding directly (CLIP-style text features), or from cross-attention between instruction tokens and visual tokens at intermediate layers? The former is similar to FlashVLM's shallow cross-attention; the latter is closer to surf's contrastive patch scorer. The mechanism determines whether CogVLA is a strong baseline or a different architecture family.
- DynaFLIP adds proprioception as a training signal for action-relevant visual features — does the proprioceptive signal distinguish contact-relevant regions (where the gripper is, what it's touching) from task-relevant regions (the target object, the goal state)? If these two are conflated in the dynamics pretraining objective, it won't isolate what surf's structural prior is supposed to capture.
- SpecPrune-VLA's "global context" is aggregated across action steps — is this a learned history encoding (like AVA-VLA's visual memory buffer) or a fixed heuristic (e.g., attention pattern averaging across the last K steps)? If learned, it raises the same circular-dependency question as the selector.
