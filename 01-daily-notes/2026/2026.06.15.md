### digest (auto)

**core**
- [Selective Perception for Robot: Task-Aware Attention in Multimodal VLA](https://www.semanticscholar.org/paper/ef8997feb9758a7f1bea4eeac1f1a6c27f5a39b6) — Son, Lee, Choi, Ko, Lim · arXiv
  This is @sonSelectivePerceptionRobot2026, on the reading list since 5/24 and in [[questions]] (view-level routing vs. patch-level). Task-aware attention that selects from multi-view inputs based on task relevance, discarding task-irrelevant background — the same selective-processing thesis as surf but at view granularity. The open question from [[questions]]: does the patch-level objection to routing networks apply at view level? Son et al. is the empirical test case. Read to determine scope: if their task-aware selection operates only at view level, the patch-level case is still open for surf to claim.

- [AC²-VLA: Action-Context-Aware Adaptive Computation in VLA Models for Efficient Robotic Manipulation](https://www.semanticscholar.org/paper/04cf2c44076c84f82673b2ca901f29ec46c7a3e3) — Yu, Wang, Li, Li, Zhu · arXiv Jan 2026
  This is the "AC²-VLA (Yu et al., arXiv Jan 2026; citekey TBD)" entry in [[reading]] and [[synthesis/adaptive compute]]. Uses action context to identify structured redundancies across temporal, spatial, and depth dimensions simultaneously — the most principled formulation among the four adaptive-VLA papers (ElegantVLA / VLA-ATTC / BRPO / AC²-VLA). Key question: does "action context" mean the previous action token embedding, or a learned prediction of which spatial regions the next action will contact? The answer determines whether AC²-VLA and surf's contrastive scorer are targeting the same latent signal.

- [Look, Focus, Act: Efficient and Robust Robot Learning via Human Gaze and Foveated Vision Transformers](https://www.semanticscholar.org/paper/8f7985729b7c0f0f365839dcfcf0474d5a9f6c01) — Chuang, Lee, Gao, Zou, Soltani · arXiv
  This is @chuangLookFocusAct2025 from [[reading]] (flagged for [[eval/gaze]] data availability check). Continuous human gaze + foveated ViT reduces visual processing while improving manipulation robustness; provides a concrete human gaze supervision pipeline for robot learning. The data-availability question from [[reading]]: does Chuang's gaze dataset complement or replace DAA as the gaze supervision signal for [[eval/gaze]]? If they release gaze-annotated manipulation demonstrations, this changes the evaluation baseline.

**adjacent**
- [RATS! Patches Talk Through Registers: Emergent Parts in Register Attention Transformers](https://arxiv.org/abs/2606.14701v1) — Yang, Neskovic, Seheult, Han, Bhattad, Yuille, Wang · arXiv Jun 2026
  Proposes RATS: ViTs with register tokens where registers learn emergent compositional part-level structure (head, wings, talons) without explicit supervision, using only self-supervised training. Part identity persists across instances of the same category. This is a supervision-free structural prior that operates at part level rather than edge/line level — if registers in the backbone already capture part-level affordance-relevant structure, surf's geometric prior (line drawings) may be an expensive approximation of what register attention provides for free. Read before claiming line drawings add signal beyond what registers already capture.

- [Gaze Heads: How VLMs Look at What They Describe](https://arxiv.org/abs/2606.14703v1) — Gandikota, Bau · arXiv Jun 2026
  Identifies a small set of attention heads in VLM language backbones ("gaze heads") whose attention tracks the image region the model is currently describing. Found via simple correlation score; can be manipulated to redirect VLM attention. Directly relevant to [[eval/gaze]] and the contrastive scorer design: if VLM-internal gaze heads track task-relevant image regions during instruction following (not just during description generation), they are a zero-shot patch importance signal that requires no gaze data, no contrastive training, and no geometric prior — a strong baseline surf must beat.

- [One Layer's Trash is Another Layer's Treasure: Adaptive Layer-wise Visual Token Selection in LVLMs](https://www.semanticscholar.org/paper/65e16ddacc0629277ee1aa176ef3288ccfba21c8) — Chen, Zhang, Zong et al. · arXiv
  Finds that tokens pruned as unimportant at one LVLM layer may be critical at a later layer; proposes keeping discarded tokens in a deferred reservoir that later layers can reactivate. Directly addresses the hard-pruning fragility concern from [[claims/iterative selection]] and the Reroute risk (carried from 6/11): the reservoir design is a concrete architectural fix that avoids hard-pruning collapse without requiring iterative selection passes. If this design generalizes to VLA backbones, it resolves surf's most persistent architectural concern.

**horizon**
- [Event-Grounded Sparse Autoencoders for Vision-Language-Action Policies](https://www.semanticscholar.org/paper/1b1a40c218d97bb68f8ebea2ad7f5ca52f1347c4) — Jin, Chatterjee, Kumar, Paleja · arXiv
  Applies sparse autoencoders (SAEs) to VLA policy hidden representations and grounds SAE features in manipulation events (grasp, place, approach). Builds mechanistic interpretability tools adapted for action outputs. Horizon: if VLA hidden representations decompose into sparse, event-grounded directions, surf's contrastive scorer should align with those directions rather than generic language-image similarity — provides a principled target for what "task-relevant" means in VLA latent space.

- [PosPrune: Visual Token Pruning with Positional Bias Correction for Efficient Large Vision-Language Models](https://www.semanticscholar.org/paper/608a7b556b1a5dad9371eae6afea8f8675028783) — Wang, Li, Yin, Liu, Wang · arXiv
  Identifies and corrects a systematic positional (center) bias in VLM token pruning: CLS-attention-based pruning preferentially retains center tokens regardless of content. Proposes a positional bias correction to make selection content-driven. Directly addresses the center-bias risk carried from 6/12 ([[claims/structure helps]] center-bias confound): if surf's line-guided selection inherits the same center bias as CLS-based methods, PosPrune's correction is the required ablation control. The bias correction is applicable to any scorer, including surf's contrastive design.

## questions

- Son et al. (Selective Perception) granularity: does their task-aware attention operate at view level (select which camera), patch level (select which spatial region within a view), or both? The [[questions]] entry asks whether the patch-level routing objection applies at view level — read methods section to answer this before citing as evidence for or against surf's patch-level approach.
- AC²-VLA action context definition: does "action context" mean the previous action embedding (past-looking), a predicted contact region (future-looking), or the task language embedding? If past-action, AC²-VLA's mechanism is closer to SpecPrune-VLA (temporal consistency) than to surf's language-geometric contrastive design.
- RATS register structure: do RATS register tokens capture manipulation-relevant parts (gripper contact surfaces, object handles, affordance regions) or general visual parts (eyes, feathers, wheels)? The answer determines whether register attention transfers from classification to manipulation tasks as a structural prior.
- Gaze Heads VLM generalization: Gandikota & Bau study description generation (not instruction following). Do VLM gaze heads track task-relevant image regions when the model is following a manipulation instruction (as in RT-2 / OpenVLA)? If so, they're a zero-shot patch importance signal competitive with contrastive training.
