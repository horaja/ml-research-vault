## papers

### digest (auto)

**core**
- [Active-O3: Empowering Multimodal Large Language Models with Active Perception via GRPO](https://www.semanticscholar.org/paper/7e40accfc7ae38cea9daa21ff958cedaa7140830) — Zhu et al. · arXiv May 2026
  Trains MLLMs to make active zoom/crop decisions end-to-end via RL (GRPO), using verifiable rewards from a benchmark suite covering open-world and domain-specific active perception. Directly answers surf's open question from [[questions]]: "Should adaptive compute be trained as a policy, a threshold rule, or a value-of-information objective?" Active-O3 says: RL policy with verifiable spatial rewards. Compare this training approach against surf's contrastive selector design — if GRPO converges well, it's an alternative to the contrastive objective.

- [PosPrune: Visual Token Pruning with Positional Bias Correction for Efficient Large Vision-Language Models](https://www.semanticscholar.org/paper/608a7b556b1a5dad9371eae6afea8f8675028783) — Wang et al. · AAAI 2026
  Training-free two-stage token pruning: coarse global relevance pass, then Asymmetric Region-aware Pruning (ARP) that explicitly up-weights peripheral tokens to counteract center bias. The ARP strategy is the methodological reference surf needs for designing center-bias-controlled baselines in [[claims/structure helps]]. If line-guided selection is also center-biased (likely, since manipulation scenes have center-heavy objects), ARP-style correction should be a required component of the comparison design.

- [FlashVLM: Text-Guided Visual Token Selection for Large Multimodal Models](https://www.semanticscholar.org/paper/4fddca70a71254c39d619c4f53727b00b4fea9fe) — Cai et al. · arXiv Dec 2025
  Language-conditioned token selection achieving "beyond lossless compression" — slightly outperforms the unpruned LLaVA 1.5 baseline while pruning up to 77.8% of visual tokens. The beyond-lossless result is the key claim for [[claims/language helps]]: task conditioning doesn't just reduce compute, it selects *better* tokens. Directly instantiates the language-conditioned selection mechanism surf proposes. Read before finalizing the language-conditioning experiment design.

**adjacent**
- [Coarse-to-Control: Action-Token Planning for Vision-Language-Action Models](https://arxiv.org/abs/2606.07107v1) — Wu et al. · arXiv Jun 2026
  Plan-execute VLA: first generates a compact coarse action-token sequence as a trajectory plan, then executes with fine-grained control. Adjacent to [[claims/iterative selection]] — the coarse-then-refine commitment structure is the same design pattern surf proposes for visual token selection (cheap prior → task-conditioned refinement). Different axis (planning vs. spatial), but the two-stage framing and rationale are structurally analogous.

- [RhinoVLA Technical Report](https://arxiv.org/abs/2606.07383v1) — Huixi Intelligence et al. · arXiv Jun 2026
  Edge-deployment VLA report identifying VLM visual and context tokens as the dominant latency source: GEMM computation grows linearly with token count at fixed model dimension. Practical confirmation that visual token reduction is the correct efficiency target for VLA deployment — not layer count or attention span. Relevant to surf's compute budget motivation and the framing of efficiency gains.

**horizon**
- [The Model Knows Which Tokens Matter: Automatic Token Selection via Noise Gating](https://www.semanticscholar.org/paper/cad7509eb8a3cb2b074c33e7f57c242853e2b149) — He et al. · arXiv Mar 2026
  Attaches a lightweight Scorer + Denoiser to a frozen VLM and trains with standard next-token prediction only — no auxiliary losses, no explicit task labels. Claims the model's own NTP gradient signal is sufficient to identify which tokens matter. The key question for surf: does NTP-only token selection capture the structural/action-relevant spatial signal that surf's line-prior + contrastive selector is designed to inject? If NTP alone achieves the same spatial selectivity, surf's explicit contrastive supervision may not add information — but if NTP selection is semantically diffuse (which "When Attention Collapses" from 6/4 suggests it is in later layers), surf's structural prior still has a role.

- [TEVI: Text-Conditioned Editing of Visual Representations via Sparse Autoencoders for Improved Vision-Language Alignment](https://arxiv.org/abs/2606.07451v1) — Mahajan et al. · arXiv Jun 2026
  Uses sparse autoencoders to identify and edit visual representation features that cause VL misalignment (images carry more information than captions describe); text-conditioned editing steers image embeddings toward the relevant language-described subspace. Horizon: pairs with Jin et al. (6/6) SAE probing of VLA layers. If SAE probing identifies which intermediate layers carry action-relevant spatial features (Jin et al.), TEVI's SAE editing approach suggests how to align those features toward instruction. Together they outline a probe-then-edit pipeline for surf's contrastive alignment experiment.

## questions

- Active-O3 training: does GRPO-trained active perception require dense reward signal per zoom step, or does a sparse end-task reward suffice? If sparse rewards work, this is tractable for surf's manipulation setting where per-token reward is unavailable.
- PosPrune ARP: is surf's line-guided selection center-biased? If line drawings cluster near center (manipulation task objects typically are), then line-prior selection is confounded with center bias and ARP-style correction is mandatory for a fair comparison.
- FlashVLM "beyond lossless": is the beyond-lossless result robust across diverse instruction types, or does it rely on high visual-instruction correlation in VQA benchmarks? Would it hold in robot manipulation where instructions are short and spatial reference is implicit?
- AutoSelect NTP-only: does noise-gated NTP selection preserve edge/peripheral structural tokens (surf's line prior selects these), or does it preferentially select semantically salient center tokens?
