This is empirical support for surf's foundational premise — visual encoding is the compute-critical pathway in VLA — but it also raises a harder question: if standard manipulation benchmarks exert limited pressure on language grounding, the tasks where surf's contrastive scorer adds value must be specifically those that require instruction disambiguation. This changes the scope of the evaluation claim.

## questions

- STS first stage vs. surf's geometric prior: does "When Attention Collapses" STS repulsion-based sampling enforce structural diversity using spatial spread alone (any spatially diverse token set), or does it weight tokens by structural feature density (edge-rich, line-dense regions)? If spatial spread alone, surf's line-drawing prior supplies the missing structural signal — this is the differentiation axis. If it uses structural features, surf needs a stronger claim.
- Drop-Then-Recovery + contrastive scorer scope: if language backbone blocks are redundant for standard manipulation, does their removal degrade performance specifically on tasks requiring spatial instruction-following (e.g., "pick the red cup to the left")? If yes, the degradation zone is exactly where surf's contrastive scorer operates. Testable on the DTR paper's setup.
- Step-wise language sensitivity × geometric prior: which manipulation steps (approach, contact initiation, tool engagement, release) are language-sensitive vs. agnostic? Do language-sensitive steps cluster at geometrically active moments (high line density, contact zones)? If yes, temporal language sensitivity and spatial geometric structure are proxies for the same signal.
- TDANN stream specialization: does the topographic locality objective in Finzi et al. produce V4-like shape/texture column organization without explicit shape-bias supervision? If yes, the V4 map used as surf's prior may be obtainable from a topographic ViT trained on naturalistic data — removing the need for the digital-twin pipeline.
