## opener

Wednesday, July 1. GSPA unread — day 18. "When Attention Collapses" (STS) unread — day 7. Contrastive alignment spec unwritten — 7+ weeks.

Today's most consequential paper: ST-Merge (Chen et al., "Fast Enough to Act") achieves 8.3× inference speedup on π0.5 VLA at 1024×1024 resolution using training-free spatiotemporal token merging — geometrically consistent fusion using 3D spatiotemporal coordinates, with positional encoding correction via rotational position code re-evaluation. This expands the competing-baseline tier to three training-free VLA efficiency methods: DepthCache (depth structure), SpecPrune (action granularity), ST-Merge (spatiotemporal redundancy). The differentiation question for surf's spec narrows: does line-geometric structural density predict task-relevant patches better than spatiotemporal redundancy? ST-Merge is geometry-aware but geometry-of-proximity (spatially nearby, temporally similar tokens merge); surf's prior is geometry-of-structure (line/shape density marks patches regardless of inter-frame redundancy). Spec differentiation table now requires a row for ST-Merge.

ERA (Wang et al.) identifies "Attention Logit Collapse" — dropping tokens distorts softmax normalization via their key vectors, degrading attention quality in remaining tokens. ERA fixes this with entropy-guided pruning + rectified attention. This is a design constraint for surf: if the geometric prior drops tokens before LLM attention runs, surf may induce the same collapse. Either surf needs attention rectification in the token-reduction path, or the prior should be scoped to drop only low-entropy tokens (per ERA's criterion). Read alongside the layer-wise token importance finding from ALVTS (6/28) before writing the spec architecture section.

Third notable: Cortinovis et al. propose action as a topographic organizing principle of visual cortex alongside animacy and real-world size — if action-selectivity gradients co-locate with shape/texture columns in V4-adjacent regions, [[claims/v4 relevance]] extends from structural to action-predictive features directly.

## questions

- ST-Merge vs. structural prior: does spatiotemporal redundancy (tokens that merge because they are geometrically close + temporally similar) co-locate with structural sparsity (flat, featureless patches with low line density)? If yes, ST-Merge and surf's prior largely agree on which tokens to drop, and the differentiation must be on what they *disagree* about (tokens that are structurally dense but temporally stable, or tokens that are structurally sparse but temporally dynamic). If no, they are complementary, and surf should propose combining them.
- Attention Logit Collapse × surf's design: at what stage does surf's geometric prior apply? If the prior drops tokens before the LLM attention runs (input-stage pruning), attention logit collapse is a real risk. If surf's prior operates by downweighting tokens in the scorer stage (before selection, after encoding), the LLM attention sees all tokens and the collapse is avoided. Which architecture is implied by [[experiments/contrastive alignment]] as currently framed?
- Dense CoT sub-task steps × geometric criticality: which manipulation sub-tasks (approach, contact initiation, tool engagement, release) require geometric scene perception, and which rely primarily on semantic object-class signals? If geometric criticality concentrates in contact-initiation and tool-engagement steps, surf's structural prior adds value specifically at those steps — aligning with "When Does Language Matter" (6/29) step-wise language sensitivity.
- RL for spatial selection: can surf's selection decisions be framed as a policy over patch subsets, with RL reward defined as task success on the selected subset? What prevents this from collapsing to always-select-all-patches as a degenerate solution?
- Action topography × V4 shape columns: in Cortinovis et al.'s topographic DANN, do action-selective units emerge in the same spatial region as shape/texture-selective units? If yes, V4's shape/texture column organization is an instantiation of action-relevant topography — closing the loop between [[claims/v4 relevance]] and [[claims/structure helps]].
