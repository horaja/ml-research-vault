## opener

Saturday, July 18. Contrastive alignment spec unwritten — 12 weeks. GSPA unread — day 35. "When Attention Collapses" (STS) unread — day 24.

Two papers in today's digest directly sharpen the spec. **Probing-Based Sequential Decision Making Under Cognitive Limitations** (Kumar et al., ISIT) formalizes exactly surf's budget problem: an agent with limited processing capacity (= visual budget) must adaptively probe data (= inspect tokens) to test a binary hypothesis (= is this patch action-relevant?) under cost constraints. This is the formal ground surf has been missing for [[claims/iterative selection]]. The "cognitive impairment in data processing" term maps directly to surf's per-step token budget. If surf's iterative selector can be cast as an active SHT with probing, this provides both a stopping criterion and a principled way to compare with one-shot selection under the same budget.

**History-Conditioned Spatio-Temporal Visual Token Pruning for Efficient VLN** (Wang et al.) proposes a training-free mechanism that uses temporal history to prune tokens across navigation steps. This is a new competing baseline for [[claims/iterative selection]] — history-conditioned pruning without any structural prior and without contrastive training. The key diagnostic: does their history-conditioned selection co-locate with surf's line-density selection on manipulation contact regions (LIBERO)? If yes, the structural prior is redundant once temporal context is available. If no, surf's structural signal captures task-relevant geometry that history alone misses.

**DRACO** (Ukarapol et al.) adds depth as a 3D structural prior for VLA with language conditioning. Third structural competitor to [[claims/structure helps]] after DepthCache (7/9) and G³VLA (7/25 pending): depth vs. line drawings vs. 3D geometric features as the structural signal. Spec must account for this.

Write the spec.

## questions

- History-conditioned pruning × [[claims/iterative selection]]: Wang et al. prune by temporal redundancy across navigation steps. In manipulation, what is the analog of "temporal redundancy"? Across action steps, the end-effector approaches the contact region — tokens that were peripherally relevant early become centrally relevant later. Does history-conditioned pruning track this shift, or does it suppress tokens that re-enter relevance? If it suppresses re-entering tokens, surf's line-density selection (which is always structurally grounded at each step) has independent value.
- Probing-Based SHT × surf spec: Can surf's iterative selector be formally cast as an active SHT with probing? If yes, Kumar et al.'s framework gives a principled stopping rule (stop when posterior uncertainty drops below a threshold set by the visual budget). If no, what breaks the mapping — is it the continuous nature of the action space, the multi-hypothesis structure of manipulation tasks, or the fact that token "probing" isn't independent across selection steps?
- DRACO × [[claims/structure helps]]: DRACO integrates depth via 3D flow matching. Depth is a metrically correct 3D structural prior; line drawings are 2D edge features. On scenes where 3D depth and 2D line density diverge (e.g., flat textured surfaces with no depth discontinuities), which signal better predicts action-relevant regions? This is an empirical question on LIBERO workspace images.
- FoMoVLA × [[claims/iterative selection]]: FoMoVLA forecasts future visual states; surf's iterative selection adapts token sets at each step. If FoMoVLA's foresight correctly predicts which regions matter at each future step, does surf's per-step selection become a downstream implementation detail? Or does surf's structural grounding (line density anchors selection to geometry) provide stability that pure foresight lacks when prediction errors compound?
