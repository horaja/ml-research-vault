# 2026-07-05

## opener

Sunday, July 5. GSPA unread — day 22. "When Attention Collapses" (STS) unread — day 11. Contrastive alignment spec unwritten — 7+ weeks.

Today's lead: Dong et al. (arXiv 2606.11906, ACL 2026) run the first systematic multilingual evaluation of VLA language sensitivity on LIBERO, finding that VLA models are not uniformly language-responsive: sensitivity is step-wise, not global — some manipulation steps respond strongly to instruction variation, others are nearly language-blind. This directly constrains [[claims/language helps]]. Surf's contrastive scorer applied uniformly across all inference steps will waste compute (and may inject noise) at language-insensitive steps, where the patch selection signal is disconnected from what actually drives the action. Two design responses: either predict which steps are language-sensitive and activate the scorer selectively, or demonstrate that surf's geometric pre-filter is beneficial even at nominally language-insensitive steps (because structural density encodes contact geometry that matters regardless of instruction content). The second is the stronger claim and the cleaner experiment.

Second: Zhang & Bisk (arXiv 2606.23641) identify the mechanism behind instruction blindness: VLA finetuning on limited robot data narrows the loss landscape, collapsing the pretrained model's language representations. Flatness-seeking optimization (SAM-style) preserves the broad landscape and retains language sensitivity without any explicit language-conditioning module. This is the alternative repair to surf's contrastive scorer. The relationship is: if surf's scorer requires a non-collapsed language encoder to produce meaningful patch scores, then Flatness is a prerequisite, not a substitute — and the ablation table must include a "Flatness + no scorer" condition to isolate surf's contribution over passive representation preservation.

## questions

- Step-wise language sensitivity × scorer activation: if some manipulation steps are language-insensitive (Dong et al.), which step features predict sensitivity — is it the step's place in the task sequence, the action type (grasping vs. placing vs. opening), or the instruction's referential specificity? If the pattern is systematic, surf could use a learned step-sensitivity head to gate scorer activation, reducing unnecessary cross-modal scoring passes at insensitive steps.
- Flatness prerequisite × surf scorer ordering: does surf's contrastive patch scorer require a non-collapsed language encoder to produce meaningful scores? If yes, the training pipeline must be: (1) pretrain with flatness-seeking finetuning, (2) add surf's scorer on top. If no — if the scorer can recover language sensitivity from a collapsed encoder — then Flatness and surf are truly competing repairs, and the ablation tells which is more efficient.
- Moving Eye view diversity × patch selection orthogonality: at test time (fixed camera), surf's patch selector applies within a single view. If spatial generalization failures (Moving Eye) stem from training-time view poverty, does adding surf's patch selection at test time help at all? Or does surf only add value when the model has already seen sufficient view variation during training?
- ACID inverse dynamics × surf patch criterion: if ACID's inverse-dynamics feasibility check identifies "action-consistent" scene regions, and surf's structural prior identifies geometrically dense regions, do these two region sets co-locate on manipulation scenes? If yes, structural density is an efficient proxy for action-consistency. If no, the structural prior misses some action-consistent regions and catches some non-action-consistent ones — the mismatch is quantifiable and is the core ablation for [[claims/structure helps]].
- RL self-reflection training × surf selection-value critic: Tang et al. use RL to train error-correction via re-attention; the reward signal is task success after reflection. Can the same RL signal train surf's patch reselection — reward for selecting a different patch set when the previous selection leads to a failed action, or a lower-quality predicted trajectory? This would convert surf's iterative selection from a heuristic to a learned policy.
