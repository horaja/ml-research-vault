## papers

### digest (auto)

**core**
- [G³VLA: Geometric inductive bias for Vision-Language-Action Models](https://arxiv.org/abs/2606.24472v1) — Peng, Zhao, Habuda, Pham, Zhu, Nguyen Le, Abu-Dakka, Guo · arXiv (Jun 2026)
  Proposes a camera-aware geometric module that re-grounds VLA visual tokens in calibrated 3D camera geometry rather than 2D image coordinates — a mismatch G³VLA identifies as especially severe in multi-camera setups where known intrinsics/extrinsics are currently ignored. This is the first explicit "geometric inductive bias for VLA" paper in the digest. The critical assessment for surf: surf's structural prior targets 2D within-image features (line drawings, edge structure, shape); G³VLA targets 3D cross-view camera geometry. These could be orthogonal (different inputs, different inductive biases) or competing (both claim "geometry helps VLA"). Read alongside GSPA before spec novelty section is written.

- [Supervise What Survives: Geometry-Guided VLA Adaptation from Synthetic Robot Videos](https://arxiv.org/abs/2606.24448v1) — D. Chen, Y. Chen, Huang, Cao, Gao, Shou · arXiv (Jun 2026)
  Generated robot videos produce correct geometry (spatial trajectory = "where") even when pixel appearance is wrong; supervises VLA adaptation using only the geometric signal rather than pseudo-actions from synthesized pixels. Directly supports [[claims/structure helps]]: if geometry encodes "where" and "where" is the load-bearing task signal, then structural patches (the geometrically dense regions selected by surf's line prior) are the right target. Counter-question: if low-level geometry (depth/trajectory) already captures "where," does surf's 2D line-drawing prior add anything beyond what a depth or optical-flow prior provides?

- [InSight: Self-Guided Skill Acquisition via Steerable VLAs](https://arxiv.org/abs/2606.24884v1) — M. Wang, Osterberg, Tian, Shorinwa, Wu, Schwager · arXiv (Jun 2026)
  Makes VLAs steerable at primitive-action level ("move gripper to bowl," "lift," "pour") by automatically segmenting demonstrations into labeled primitives; at inference, generates sub-goal sequences that steer the VLA to acquire new skills without additional demonstrations. Relevant to [[claims/iterative selection]]: if VLA primitives naturally correspond to visual state transitions (approach, contact, grasp release), InSight's primitive-level structure identifies *when* visual refinement matters — a potential operational "when-to-refine" criterion complementary to KEMO's kinematic event detector.

**adjacent**
- [ActiveScope: Actively Seeking and Correcting Perception for MLLMs](https://arxiv.org/abs/2606.24292v1) — Y. Wang, Bi, Sun, Shen, Qi, S. Wang, Huang · arXiv (Jun 2026)
  Fine-grained perception for high-resolution MLLMs via active seek-and-correct: proposes candidate regions, verifies whether each satisfies the query, corrects if not. Core failure mode diagnosed: "Contextual Dominance" — salient distractors overwhelm target attention, causing failed localization when multiple targets exist. Adjacent to surf's active perception framing: Contextual Dominance is the visual analogue of instruction blindness (the wrong region captures attention). ActiveScope's verify-and-correct loop is a test-time counterpart to surf's iterative selection + stopping criterion, without contrastive training.

- [World Value Models for Robotic Manipulation](https://arxiv.org/abs/2606.24742v1) — Z. Wang, Li, Cui, Gao, Zhan, Yu, Ma · arXiv (Jun 2026)
  Generalist value models with deep temporal understanding for robot policy learning from mixed-quality data: learns to ground current belief using history and plan over future outcomes. Adjacent to surf's RL/efficiency module: a value model that estimates uncertainty about current task state is a principled criterion for "how much visual information is needed now" — which is exactly surf's adaptive budget question. If WVM's temporal grounding mechanism identifies high-uncertainty steps, those are the steps where surf's visual refinement is most valuable.

- [Beyond Monotonic Progress: Retry-Supervised Value Learning for Robot Imitation](https://arxiv.org/abs/2606.24633v1) — (Zhihao Wang et al.) · arXiv (Jun 2026)
  Correction behaviors in demonstrations (imprecise grasps, unstable contact, repeated attempts) encode task-structure information ignored by monotonic-progress-assumption value models; learns value from retries. Adjacent to surf's structural prior: if correction/retry regions in demonstrations cluster at geometrically complex areas (object boundaries, grasp contact zones) — the same regions surf's line prior selects — that is independent weak-supervision evidence for [[claims/structure helps]]. Testable: where do retries occur in surf's manipulation scenes?

**horizon**
- [Can Scale Save Us From Plasticity Loss in Large Language Models?](https://arxiv.org/abs/2606.24752v1) — (authors TBD from abstract) · arXiv (Jun 2026)
  Studies plasticity loss (inability to learn new information after prior training) in modern transformer LLMs at scale. Horizon connection: "Don't Blind Your VLA" (6/21 digest) showed VLA fine-tuning degrades visual representations — plasticity loss is the mechanistic account of that same phenomenon. If scale alone does not prevent plasticity loss, explicit structural interventions become more important: surf's geometric prior may serve a dual function as both a feature selector and a plasticity-preserving constraint that keeps high-salience structural features active during VLA fine-tuning.

- [AVOC: Enhancing Hour-Level Audio-Video Understanding via Retrieval-Inspired Token Compression](https://arxiv.org/abs/2606.24286v1) — Y. Chen, Tan, Yu, Y. Wang, Cheng, Guan, Jiang, Li, Zhu, Song · arXiv (Jun 2026)
  Retrieval-inspired token selection for long-form audio-video: uses a query to retrieve the most task-relevant tokens across a long temporal context, without discriminative training. Horizon framing for surf: retrieval is an alternative paradigm to contrastive training for task-conditioned token selection. If a retrieval baseline without fine-tuning achieves comparable patch selection to surf's trained contrastive scorer, it simplifies the training pipeline — and makes the contrastive scorer's value contingent on outperforming retrieval.

## questions

- G³VLA 2D vs. 3D scope: Does G³VLA's camera-aware geometric module operate on the same visual features as surf's line-drawing structural prior, or are they genuinely orthogonal (3D camera geometry vs. 2D shape features)? If G³VLA addresses multi-camera 3D geometry while surf addresses within-image 2D structure, they compose without competing. If G³VLA's 3D re-grounding subsumes the gains from surf's 2D prior (by providing a richer geometric prior than line drawings), the 2D structural prior claim weakens. Read the G³VLA methods section.
- Supervise What Survives geometry axis: Does the paper's "geometry" signal overlap with surf's structural features (line drawings, shape), or is it specifically depth/trajectory? If geometry = depth + optical flow, surf's line-prior is a different geometric axis (2D shape structure) that the paper's approach does not cover.
- InSight primitive-to-contact alignment: Do InSight's automatically segmented primitives segment at contact-relevant transitions (grasp initiation, object release, tool engagement), or at coarser linguistic transitions? If primitive boundaries align with contact transitions, InSight's primitives provide a language-natural definition of surf's "when to refine."
- Beyond Monotonic Progress correction zones: In manipulation demonstrations, do correction/retry events (unstable grasps, misaligned contacts) cluster at structurally complex regions — the same regions surf's line prior selects? This is directly testable on surf's manipulation data.
