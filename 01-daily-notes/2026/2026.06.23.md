## papers

### digest (auto)

**core**
- [Flatness Preserves Instruction Following in Vision-Language-Action Models](https://arxiv.org/abs/2606.23641v1) — Zhang, Bisk · arXiv (Jun 2026)
  "Instruction blindness" — VLA finetuning on limited data causes policies to ignore language and take visual shortcuts. Fix: SAM during finetuning, +60% instruction following across simulation and real-world benchmarks, no data augmentation or architecture change. From CMU. Two readings are possible: (a) structural priors are unnecessary if flatness alone recovers language grounding, or (b) SAM recovers *temporal* instruction following but not *spatial* instruction grounding — and surf's contrastive scorer adds the spatial dimension. Which reading is correct depends on whether "flatness" affects what the model attends to, or only when it executes.

- [Learning to See While Learning to Act: Diffusion Models for Active Perception in Robot Imitation](https://arxiv.org/abs/2606.23625v1) — Wang, Saxena, Cheng, Koga · arXiv (Jun 2026)
  See2Act: jointly learns viewpoint selection and action prediction from demonstrations. Camera poses are anchored to keyframe actions in offline data; at test time, action denoising is coupled with viewpoint refinement. +34% over prior methods on RLBench under occlusion; zero-shot sim-to-real with 50 demonstrations. This is the active-perception paper most directly relevant to surf's open questions (questions.md, "active perception" section) — it formalizes the coupling principle surf proposes at viewpoint granularity. Surf's structural prior operates within-frame (patch selection); See2Act operates across views (camera-pose selection); the gap between them is an explicit open question. If See2Act's viewpoint criterion aligns with surf's patch criterion, they compose naturally.

- [KEMO: Event-Driven Keyframe Memory for Long-Horizon Robot Manipulation with VLA Policies](https://arxiv.org/abs/2606.23589v1) — Zeng, Ye, Chen, Shentu · arXiv (Jun 2026)
  Event-driven keyframe selection for VLA memory: detects task-relevant state transitions via robot kinematics + visual filtering, encodes selected keyframes as compact memory tokens, fuses with current features via cross-attention and gated residual. +23.6% Task Success Rate, +34.1% Stage Completion over π0.5 (memory-free baseline) on real-world dual-arm tasks. Directly addresses [[claims/iterative selection]]: event detection is the temporal analogue of surf's "when to refine" stopping criterion. Key question: does KEMO's kinematic + visual filter converge on the same keyframes as the language-sensitive steps identified by "When Does Language Matter?" If kinematic events ≠ language-sensitive steps, kinematic filtering is insufficient and language conditioning adds signal. If they coincide, KEMO's approach subsumes surf's temporal selection.

**adjacent**
- [HiLo-Token: Input-Adaptive High-Low Frequency Token Compression for Efficient Image Editing](https://www.semanticscholar.org/paper/b45e04db4b850561e673b009fe137ea0b37b636b) — You, Nitzan, Zhang, Gong · arXiv (Jun 2026)
  Token budget split by spatial frequency: all tokens in the high-frequency editing region (dilated mask) retained; outside, high-frequency tokens (spatial gradient) kept for local detail, 16x downsampled low-frequency tokens for global structure. 3.13x speedup on A100 for small-mask tasks without quality regression, in diffusion transformers for image editing. Not robotics. Relevant to surf as a practical validation of the structural-prior token allocation hypothesis — more tokens to high-frequency (structural/edge) regions, fewer to low-frequency (semantic/smooth) regions. The frequency decomposition maps closely onto surf's line prior: lines are high-spatial-frequency content. If this allocation is optimal in image generation, the same argument applies in visual feature extraction for VLAs.

- [When Does Language Matter? Multilingual Instructions Reveal Step-wise Language Sensitivity in Vision-Language-Action Models](https://www.semanticscholar.org/paper/094964f7e202bcf536d92558d0e490ed231c8c34) — Dong, Han, Niu, Zhu · arXiv (Jun 2026)
  First systematic multilingual VLA evaluation (LIBERO × 10 languages): 30–50% success rate drop under non-English instructions. Mechanistic finding: language influence is step-wise non-uniform — certain steps dominate failure, others are language-agnostic. Fix: inference-time step-wise representation alignment recovers much of the gap. Required read for the spec's language-conditioning design. Directly constrains surf's contrastive scorer: if language sensitivity is step-sparse, a step-adaptive scorer (active only at high-sensitivity steps) is more efficient and potentially more effective than a step-uniform one. This is the step-adaptive design constraint referenced in the spec blocklist since 6/20.

**horizon**
- [Kamera: Unified Position-Invariant Multimodal KV Cache for Training-Free Reuse](https://arxiv.org/abs/2606.23581v1) — Ma, Eitzinger, Koestler, Wellein · arXiv (Jun 2026)
  KV cache reuse for multimodal agents: identifies that naive reuse loses cross-chunk conditioning, a low-rank signal concentrated in deep LLM layers. Fix: a small training-free low-rank patch stored per cached chunk, recovering multi-hop binding accuracy. Adds a new data point on the layer-depth question (6/19–6/22 convergence): cross-chunk visual binding is deep-layer phenomenon, consistent with contrastive scorer placement at intermediate-to-deep LLM layers. The "conditioning is low-rank, deep" finding is a sixth convergent signal on where surf's scorer should extract features.

- [dVLA-RL: Reinforcement Learning over Denoising Trajectories for Discrete Diffusion VLA Models](https://arxiv.org/abs/2606.23623v1) — Wu, Liu, Shen, Han · arXiv (Jun 2026)
  RL training for discrete diffusion VLAs (masked generative action modeling): models the denoising chain as an MDP with trajectory-level probability objective. Variable denoising steps per task complexity. 99.7% on LIBERO, +30.6% over SFT on RoboTwin. Horizon connection to surf: the variable-step denoising = implicit adaptive compute. If RL-trained VLAs learn to adaptively allocate denoising compute by task complexity, the same MDP framing could be applied to surf's visual budget allocation — "how many patches to select" as a policy over denoising steps. The trajectory-level RL objective is closer to surf's adaptive compute framing than per-step supervised selection.

## questions

- See2Act viewpoint vs. patch granularity: See2Act's viewpoint selection (camera pose) and surf's patch selection (within-frame) operate at different granularities of the same visual allocation problem. Does See2Act's implicit where-to-look criterion (camera poses anchored to keyframe actions) converge on the same spatial content as surf's structural prior when both are applied to the same scene? If yes, they are parallel solutions at different granularities and can be composed. If no, viewpoint and patch selection are orthogonal axes — surf needs to address both.
- Flatness-VLA spatial vs. temporal instruction following: SAM recovers temporal instruction following (which phase to execute) by 60%+, but does it also recover *spatial* instruction grounding (directing visual attention to the right region)? If SAM recovers spatial grounding, surf's structural prior is less necessary for the language-conditioning claim. If SAM recovers temporal but not spatial grounding, surf's contrastive scorer fills the remaining gap.
- KEMO × When Does Language Matter? alignment: KEMO detects events using kinematics + visual filtering without language conditioning. Does the detected event coincide with the language-sensitive steps from Dong et al.'s step-wise analysis? If yes, kinematic event detection is a cheap proxy for language-sensitive step identification. If no, the language signal contains information that kinematic features miss — and surf's language-conditioned scorer adds non-redundant signal.
- HiLo-Token: In robot manipulation scenes (not image editing), do the high-frequency spatial-gradient tokens correspond to contact/affordance regions (tool-object boundary, gripper-contact surface) or to object boundaries more broadly (including non-action-relevant background edges)? If high-frequency tokens ≠ action-relevant tokens, surf's structural prior must be more specific than frequency-based selection.
- dVLA-RL denoising steps: does the variable denoising step scheduling in dVLA-RL learn to use more steps for contact-rich tasks and fewer for reach-only tasks, or does task complexity not correlate with contact-richness? The answer constrains how well a denoising-as-MDP framing transfers to surf's spatial selection policy (which is contact-motivated).
