If policies pretrained on language-action pairs alone transfer better than policies trained on visual shortcuts, the question becomes not "do visual patches help" but "when and why do they help beyond language alone." The action-granularity controller — coarse vs. fine-grained actions get different compute budgets — is the behavioral analog of surf's adaptive compute module, using end-effector speed rather than language-conditioned geometric structure as the criterion.

### digest (auto)

**core**
- [LA4VLA: Learning to Act without Seeing via Language-Action Pretraining](https://arxiv.org/abs/2606.27295v1) — T. Lin, Y. Du, Y. Mao et al. · arXiv
  Pretrains VLA on language-action pairs without any visual input; resulting priors transfer to visual settings and improve success by up to +45pp real-world, +17.8pp sim over no-pretraining baselines. This is surf's visual necessity argument in its sharpest form: if language-action priors capture reusable manipulation skills without scene-specific visual cues, surf must specify which conditions require geometric visual patches that language alone cannot handle. Must appear in spec motivation alongside the visual-shortcut failure mode diagnosis.

- [One Layer's Trash is Another Layer's Treasure: Adaptive Layer-wise Visual Token Selection in LVLMs](https://www.semanticscholar.org/paper/65e16ddacc0629277ee1aa176ef3288ccfba21c8) — Y. Chen, K. Zhang, Z. Zong et al. · arXiv
  Empirically shows different transformer layers focus on different visual regions; static token pruning causes premature information loss when dropped tokens become critical at later layers. ALVTS routes tokens per layer (skip + reintegrate), achieving 89% compression with 96.7% accuracy retention. Directly relevant to surf's selector design: surf's geometric prior operates at the input stage, but layer-wise importance variation suggests optimal structural token coverage changes across depth. If geometric tokens are important in early layers and semantic tokens in late layers, this independently validates surf's two-stage prior-then-scorer architecture — or suggests the prior should be applied per-layer rather than once at input.

- [SpecPrune-VLA: Accelerating VLA Models via Action-Aware Self-Speculative Pruning](https://www.semanticscholar.org/paper/e5e4c5f50fef44a34d1235659e2cdb6cf3db5a6b) — H. Wang, J. Xu, J. Pan et al. · arXiv
  Training-free VLA efficiency: global temporal consistency + local attention for static per-action token reduction; layer-wise dynamic pruning; action-granularity controller (end-effector speed → coarse vs. fine-grained → pruning budget). Achieves 1.57×–1.70× speedup on LIBERO and real-world tasks. The action-granularity controller is the behavioral analog of surf's adaptive compute budget — both condition visual compute on task stage — but SpecPrune uses behavioral coarseness rather than language-conditioned geometry as the criterion. Add to spec differentiation table alongside AC²-VLA, LightVLA, and DepthCache.

**adjacent**
- [DreamVLA: A Vision-Language-Action Model Dreamed with Comprehensive World Knowledge](https://www.semanticscholar.org/paper/f374c4d9407566e349ab861c5d51720b211e37f9) — W. Zhang, H. Liu, Z. Qi et al. · arXiv
  Integrates world knowledge forecasting (dynamic, spatial, semantic) with VLA action prediction; block-wise structured attention prevents information leakage between knowledge types. Achieves 76.7% on real robot tasks, 4.44 avg length on CALVIN ABC-D. The dynamic-region-guided prediction identifies which spatial regions are task-relevant from world-state prediction — a data-driven version of surf's geometric prior operating at a higher semantic level. - [PAMAE: Phase-Aware-MoE Action Experts Towards Reliable Flow-Matching VLA Policies](https://arxiv.org/abs/2606.27144v1) — J. Yang, T. Yang, X. Chang et al. · arXiv
  Sparse MoE action experts with a phase-aware router that identifies task execution phase (approach, grasp, release, etc.) and allocates action generation accordingly; improves multi-stage task success by up to 9.2%. Directly adjacent to [[claims/iterative selection]]: PAMAE identifies when phase transitions occur at training time; surf needs a when-to-refine criterion at inference time. If PAMAE's learned phase boundaries align with surf's structural density transitions (contact zones, tool-object boundaries), they are operationally equivalent from different angles.

## questions

- LA4VLA visual necessity: under what conditions does language-action pretraining alone fail — which manipulation tasks require visual patches that language-action pairs cannot anticipate? If LA4VLA fails specifically at geometrically complex contact zones (fine manipulation, tool engagement, non-standard placement), that gap is exactly what surf's geometric prior fills. If it fails generically, the gap is task diversity, not geometric structure, and surf's claim needs reframing.
- ALVTS layer-wise token focus: does the layer-dependent visual region focus in ALVTS follow a structural-to-semantic transition (early layers: geometric/edge tokens; late layers: semantic/object tokens)? If yes, this independently validates surf's two-stage prior-then-scorer architecture. If the transition is task-specific rather than structural, surf's input-stage geometric prior is insufficient alone.
- SpecPrune-VLA action-granularity vs. surf's language conditioning: does SpecPrune's coarse-action mode (high end-effector speed → aggressive pruning) select the same tokens as surf's geometric prior selects for approach phases? Agreement would mean behavioral coarseness and geometric structure are proxies for the same signal; disagreement would mean language-conditioned geometry adds non-redundant information.
- DreamVLA dynamic regions vs. line prior: on manipulation scenes, do DreamVLA's dynamically predicted regions (where appearance changes across a rollout) overlap with surf's line-drawing-dense patches? Low overlap → complementary; high overlap → dynamic prediction implicitly learns geometric structure without explicit prior.
