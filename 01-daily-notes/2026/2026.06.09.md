## papers

### digest (auto)

**core**
- [CT-VAM: A Cerebello-Thalamic-Inspired Vision-Action Model for Efficient Visuomotor Control](https://arxiv.org/abs/2606.09572v1) — Li et al. · arXiv Jun 2026
  Proposes a compact VLA architecture that gates language out of per-step visuomotor execution: language specifies task intent once ("thalamic" routing), then fast visuomotor control runs without repeated LLM passes ("cerebellar" execution). Directly sharpens surf's open architectural question: if language is task-specification only, surf's language-conditioned selector may need to run only at episode start or on significant scene change — not every step. This would make the selector more tractable and decouple it from the per-step inference loop.

- [ProbeAct: Probe-Guided Training-Free Failure Recovery in Vision-Language-Action Models](https://arxiv.org/abs/2606.09740v1) — Zhang et al. · arXiv Jun 2026
  Attaches lightweight linear probes to VLA intermediate layers to detect distributional deviation from training — when the probe fires, a recovery controller engages. The probe methodology is the third independent line of evidence that action-relevant spatial features live in intermediate layers (alongside Jin et al. SAE (6/6) and Palacios et al. (6/4)). Directly useful for [[experiments/contrastive alignment]] layer selection: probe accuracy across layers identifies where action-predictive features concentrate before committing to a contrastive extraction point.

- [AdaTok: Self-Budgeting Image Tokenization with Quality-Preserving Dynamic Tokens](https://www.semanticscholar.org/paper/42e7fc28726cf85d8103d4dd75bb5218a136a203) — Lu et al. · arXiv Jun 2026
  Self-budgeting discrete 1D tokenizer that adapts token count per image in a single pass via Prioritized Representation Learning + Dynamic Pareto Weighting. The "self-budgeting in one pass" design is the tokenization-side analog of surf's structural-prior-guided selection: both allocate tokens adaptively based on content, but AdaTok operates at the tokenizer level before any downstream task conditioning. If AdaTok's dynamic allocation already captures structural complexity, surf's prior module becomes a task-conditioning layer on top of a reasonable content-adaptive base — not a replacement for uniform tokenization.

**adjacent**
- [From Rigid to Dynamic: Entropy-Guided Adaptive Inference for Long-Context LLMs](https://arxiv.org/abs/2606.09508v1) — Xu et al. · arXiv Jun 2026
  Classifies attention heads as "Rigid" (entropy near-zero, high selectivity) or "Dynamic" (entropy-varying, context-dependent) and assigns variable KV cache budgets accordingly. The per-head entropy signal is a training-free, attention-intrinsic budget allocation mechanism — adjacent to surf's per-patch need signal from line-prior or contrastive scoring. Compare: entropy-guided head budget (Xu et al.) vs. structural-prior patch budget (surf). If heads attending to structural patches are systematically Rigid, entropy-guided methods and structural priors may agree on what to keep.

- [Where Does the Answer Come From? Benchmarking View-Level Visual Evidence Identification in Multi-View MLLMs for Autonomous Driving](https://arxiv.org/abs/2606.09644v1) — Wang et al. · arXiv Jun 2026
  Introduces a benchmark that tests whether MLLM answers derive from the *correct* visual view — not just whether the answer is correct. The diagnostic (answer-correct ≠ evidence-correct) maps onto surf's core evaluation risk: token pruning may preserve task accuracy while destroying the spatial evidence that justifies the action. This eval framing should inform surf's evaluation design beyond top-1 accuracy; a ground-truth spatial mask (e.g., from AffordanceVLA or GIAVA's gaze) would let surf test whether selected tokens are the causally correct ones.

- [MemoryVLA++: Temporal Modeling via Memory and Imagination in Vision-Language-Action Models](https://arxiv.org/abs/2606.09827v1) — Shi et al. · arXiv Jun 2026
  Adds working memory (short-term context buffer) and hippocampal imagination (future-state prediction) to VLA. Relevant as a budget constraint: if VLA inference needs spatial tokens (surf) + temporal memory tokens (MemoryVLA++) simultaneously, the total token budget is spatial × temporal. Surf's design must either share budget with temporal context or operate upstream before the temporal memory module is engaged.

**horizon**
- [Tight Sample Complexity of Transformers](https://arxiv.org/abs/2606.09731v1) — Yang, Srebro, Li · arXiv Jun 2026
  Tightly characterizes VC dimension of depth-L transformers: Θ(LW log(TW)) where T is input sequence length. Token pruning reduces T from N to k; the log(T) factor shrinks by log(N/k). Clean theoretical framing for pruning-as-regularization: halving token count reduces the generalization bound by log(2) per layer — modest but non-trivial. The result is tight (matching lower bound), so this is not loose pessimism. Useful for related work framing if surf needs a theoretical motivation beyond efficiency.

## questions

- CT-VAM architecture: if language is a task-specification signal (set once), does surf's contrastive selector need to re-score patches at every action step, or only when instruction context changes? What triggers re-scoring in the manipulation setting?
- ProbeAct probing: which VLA layer achieves highest probe accuracy for action-relevant spatial features? Does it match Jin et al. SAE (6/6) intermediate-layer localization or differ? Cross-validate before finalizing [[experiments/contrastive alignment]] layer choice.
- AdaTok budget allocation: does AdaTok's content-adaptive token reduction correlate spatially with surf's structural prior (line-guided selection)? If they agree, structural prior is a task-agnostic content-complexity signal; if they diverge, the structural prior adds task-relevant signal AdaTok lacks.
- VC dimension result: does the log(T) reduction in sample complexity bound hold for the visual sub-sequence specifically (not the full token sequence including language tokens)? If surf prunes only visual tokens while keeping language tokens fixed, the bound reduction should be calculated over visual-token count only.
- Carried from 6/8: PosPrune ARP — is surf's line-guided selection center-biased? FlashVLM "beyond lossless" — does it hold under instruction diversity (short manipulation instructions vs. VQA)? Active-O3 GRPO rewards — sparse vs. dense reward signal for active perception?
