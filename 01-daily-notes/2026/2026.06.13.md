## papers

### digest (auto)

**core**
- [DepthCache: Depth-Guided Training-Free Visual Token Merging for VLA Model Inference](https://arxiv.org/abs/2603.10469) — Li, Ma, Ding, Zhu · arXiv Mar 2026
  Uses depth as a structural prior to partition VLA observations into near-field workspace (high detail, low merge ratio) and far-field background (aggressive merging), achieving training-free token compression with spatially differentiated merge ratios. This is the closest structural prior approach to surf in the literature: same problem (VLA token compression), same insight (structure guides selection), different signal (depth vs. line drawings). The depth partition directly answers the open question of whether structural cues beyond line drawings help — compare surf's line prior performance against DepthCache's depth prior as a baseline before claiming line drawings add anything beyond geometry.

- [LightVLA: Towards Efficient VLA Models via Differentiable Token Pruning](https://arxiv.org/abs/2509.12594) — Jiang, Jiang, Ma, Wen · arXiv Sep 2025
  First differentiable token pruning framework applied to VLA tasks, 26 cites. Pruner is task-agnostic: selects tokens by attention importance without structural prior or language conditioning. This is the baseline surf must beat or clearly differentiate from. If LightVLA's attention-importance pruner achieves good results without a structural prior, surf's structural prior claim ([[claims/structure helps]]) needs a controlled comparison against attention-importance as a baseline — not just random selection.

- [CogVLA: Cognition-Aligned VLA via Instruction-Driven Routing & Sparsification](https://arxiv.org/abs/2508.21046) — Li, Zhang, Shao, He · arXiv Aug 2025
  Instruction-driven routing and sparsification in VLA with V-L-A Coupled Attention; 56 cites. The instruction conditioning uses language semantics to route which visual tokens survive. This is surf's [[claims/language helps]] instantiated with 56 cites of evidence. CogVLA uses no structural (geometric) prior — routing is purely language-attention driven. Surf's structural-prior-plus-language framing needs to explain what geometry adds over CogVLA's pure language routing, and [[experiments/contrastive alignment]] is the experiment that would show this.

**adjacent**
- [SpecPrune-VLA: Accelerating VLA Models via Action-Aware Self-Speculative Pruning](https://arxiv.org/abs/2509.05614) — Wang, Xu, Pan, Zhou · arXiv Sep 2025
  Uses spatial-temporal consistency across consecutive VLA action steps: because frames are similar step-to-step, the previous step's token importance map guides current pruning without re-computing a full selection pass. 25 cites. This is an orthogonal structural prior — temporal consistency rather than geometric structure. The comparison matters: if temporal consistency-based pruning is as good as line-guided pruning without any geometric inductive bias, the structural prior may not be the key ingredient. Potentially a strong ablation baseline.

- [Beyond Surrogate Gradients: Fully Differentiable Token Pruning for VLMs](https://arxiv.org/abs/2605.28051) — He, Yao, Young, Xu · arXiv May 2026
  Reformulates token pruning as continuous control of token information via an Information Throttler — variance-preserving noise modulates each token conditioned on importance, bypassing Gumbel-Softmax's surrogate-gradient problem. More principled than SparseLearn's Soft Top-K approach (6/12 digest): SparseLearn adds noise to approximate Top-K during training; DiffPrune controls information flow continuously. Relevant to [[experiments/contrastive alignment]] architecture — two differentiable designs to choose between for gradient-through-selection: SparseLearn (Soft/Hard Top-K) vs. DiffPrune (Information Throttler). DiffPrune's claim that surrogate gradients are unreliable is a direct challenge to SparseLearn's design.

- [AutoSelect: Automatic Token Selection via Noise Gating](https://arxiv.org/abs/2603.07135) — He, Yang, Xu · arXiv Mar 2026
  Attaches a lightweight Scorer + Denoiser to a frozen VLM and trains with standard next-token prediction loss only — no contrastive objective, no auxiliary annotations. The capacity-constrained communication framing: maximize information in K selected tokens. 7 cites. Directly relevant to the [[experiments/contrastive alignment]] objective choice: if self-supervised noise gating achieves comparable token selection quality using only NTP supervision, surf's contrastive (language-conditioned) objective needs to show it selects task-relevant patches that NTP-supervised selection misses. This is a concrete ablation the spec must address.

**horizon**
- [TokenCarve: Information-Preserving Visual Token Compression in MLLMs](https://arxiv.org/abs/2503.10501) — Tan, Ye, Tu, Cao · arXiv Mar 2025
  Shows that MLLM performance degradation under token compression correlates with information loss in the attention output matrix; proposes a compression method that minimizes this attention-output mutual information loss. 16 cites. Provides the formal information-theoretic grounding for surf's [[claims/efficiency without collapse]]: "efficiency without collapse" needs an operational definition of what is being preserved — attention output mutual information is a concrete candidate. If surf's structural prior selection preserves more attention-output information than random or center-bias selection, it operationalizes the efficiency claim in a measurable way.

- [Real-Time Execution with Autoregressive Policies](https://arxiv.org/abs/2606.13355) — Lee, Park, You, Caciularu · arXiv Jun 2026
  Addresses real-time VLA deployment via asynchronous inference for autoregressive (non-diffusion) policies, enabling smooth trajectories and fast reactivity without waiting for full rollout completion. Horizon: surf reduces token count per step, but real-time deployment also requires async inference to decouple compute from action frequency. Even with efficient token selection, an AR VLA backbone may still bottleneck on generation latency — the two efficiency levers (token budget + async inference) are orthogonal and both needed for deployment.

## questions

- LightVLA differentiable design: does LightVLA's pruner use Gumbel-Softmax, straight-through estimator, or a different gradient approximation? The design choice determines whether DiffPrune or SparseLearn is a natural successor — or whether LightVLA's approach is already better than both.
- CogVLA language routing: does CogVLA's instruction-driven routing operate at the token level (per-visual-token language-similarity score) or at a coarser routing-network level (binary route/skip decision)? If token-level cosine similarity, the mechanism is essentially the same as surf's planned contrastive scorer — and the 56-cite baseline needs to be acknowledged in the spec.
- DepthCache depth-partition robustness: does the near-field/far-field depth partition hold when the task-relevant object is far from the robot (e.g., reaching for distant target)? If near-field = workspace is always the task-relevant partition, the structural prior is trivially correct by construction — which is either the best possible argument for structural priors or evidence that depth is measuring center bias.
- SpecPrune-VLA temporal consistency: how much of SpecPrune-VLA's efficiency gain comes from reusing previous token maps vs. from the pruning itself? If reuse is the primary gain, the method only works in sequential closed-loop inference — not in first-step or scene-change scenarios where surf's structural prior would need to operate.
- DiffPrune vs. SparseLearn: DiffPrune claims Gumbel-Softmax is unreliable because surrogate gradients don't reflect the true selection distribution. SparseLearn uses Soft Top-K + variance-preserving noise. Is SparseLearn also subject to this critique, or is variance-preserving noise a closer approximation than Gumbel? The answer changes which design to use in [[experiments/contrastive alignment]].
