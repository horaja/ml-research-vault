# 2026-07-15

## opener

Wednesday, July 15. Contrastive alignment spec unwritten — 10 weeks. GSPA unread — day 32. "When Attention Collapses" (STS) unread — day 21.

Today's digest leads with three papers that directly constrain the spec. **Visual Access Boundaries in VLM Reasoning** (arXiv today) introduces "Visual Access Sweep" — a causal masking intervention that progressively blocks generated-token attention to image-token keys across layer depth and generation time — and defines the Visual Access Boundary (VAB) as the minimal access region below which reasoning collapses. This is the mechanistic answer the STS/"Ebb and Flow" papers left open: if the VAB is shallow (image tokens needed only in early layers), surf's input-stage selection is architecturally sufficient — supply the right tokens before the boundary and later layers reason over them without revisiting image keys. If the VAB is deep or extends into late generation steps, relay-window reinsertion (STS/"Ebb and Flow" fix) is required even after input-stage pruning. Read this today alongside STS and "Ebb and Flow of Multimodal Focus" before writing the spec novelty section — the three papers together define the design space.

**What Makes a Representational Prior Work** (arXiv today) characterizes contrastive priors empirically across 188 runs: the wrong feature family (e.g., magnitude bands) blocks generalization *equivalently to a random partition*, even when the prior is coherent and learnable from that feature family. This directly constrains [[experiments/contrastive alignment]]: surf's line-density prior must be the right feature family for action-discriminative token selection, not just a structural signal that loosely correlates with task-relevant patches. The spec's ablation table needs at least one wrong-family control.

**VisCo** (arXiv today) uses the VLM backbone's own LLM encoder as the intrinsic visual compression signal — no external module, no auxiliary training objective, no contrastive scorer. The VLM itself identifies redundant tokens. Direct training-based competitor to [[claims/language helps]]: if intrinsic LLM encoding captures language-conditioned relevance as well as surf's external contrastive scorer, the contrastive training objective adds overhead without adding selection quality. The differentiation argument must be action-discriminative: surf's scorer optimizes which tokens are most informative for action prediction; VisCo optimizes which tokens the language model considers uninformative for understanding. These are not the same criterion — but the spec must say that explicitly.

The spec, STS, GSPA, and Compressor-VLA remain the four critical blockers. Read Visual Access Boundaries today. Write the spec.

## questions

- Visual Access Boundaries × surf spec: VAB defines the minimal layer/time for image token access in VLM forward passes. Does input-stage token selection (surf) satisfy the VAB constraint automatically — tokens are present before the boundary — or does restricting which tokens are present reshape the VAB itself? If sparser input forces the model to look longer to extract sufficient information, input-stage pruning may deepen the VAB and make relay-window reinsertion necessary even after surf-style selection. This is testable: compare VAB depth under dense vs. pruned token inputs.
- What Makes a Prior Work × [[experiments/contrastive alignment]] spec: line density must be validated as the correct feature family for action-discriminative selection. The paper predicts: if line density is the wrong family, the contrastive prior blocks generalization like random. What ablation verifies the family? Candidates: (a) depth gradients as structural control, (b) random patch masks as null, (c) V4 dispersity as biologically-grounded alternative, (d) VLM attention maps as language-conditioned control. Include at least one in the spec.
- VisCo × [[claims/language helps]]: VisCo identifies tokens uninformative to the LLM; surf's contrastive scorer identifies tokens discriminative for action prediction. Are these the same tokens on manipulation scenes? If yes, surf's training cost is wasted. If no, surf selects tokens VisCo compresses away — and surf's action-discriminative criterion is the value-add. This is an empirical question resolvable by comparing selected token sets on LIBERO.
- Inhibited Self-Attention × surf structural prior: ISA uses lateral inhibition architecture; surf uses line-density input bias. Do they select the same tokens? The co-location test is the same as FDT foveation (7/13) — running this test across ISA, FDT, and line density simultaneously would resolve which mechanisms are proxies for the same criterion.
- Decoupling Declarative/Procedural × surf architecture: if surf's structural prior improves declarative token selection (which regions to look at) but procedural action execution is impaired by VLA fine-tuning separately, surf may achieve its token selection goal without improving task success. Diagnostic: evaluate surf's selection quality on declarative metrics (gaze alignment, contact-region overlap) vs. procedural metrics (task success, action accuracy) independently.
