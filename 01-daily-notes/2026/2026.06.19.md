- [GSPA: Geo-Semantic Prior-Guided Attention for Long-Horizon Robot Manipulation](https://www.semanticscholar.org/paper/593fe3f7bbdcbd6169f9903fcab28e8e65316c4e) — Che, Wang et al. · Semantic Scholar
  The closest known prior work to surf's combined geometric+semantic prior claim. GSPA uses geometric AND semantic priors to guide VLA attention across long-horizon manipulation rollouts, addressing viewpoint variation and stage transitions. Must read before writing [[experiments/contrastive alignment]] related-work: the key question is whether GSPA's "geometric prior" operates at patch level or scene level, and whether it uses language conditioning. If it does both, surf's differentiation argument needs to be much more specific than "combined prior."

- [One Layer's Trash is Another Layer's Treasure: Adaptive Layer-wise Visual Token Selection in LVLMs](https://www.semanticscholar.org/paper/65e16ddacc0629277ee1aa176ef3288ccfba21c8) — Chen, Zhang et al. · Semantic Scholar
  Different visual tokens are best pruned at different ViT layers rather than at a single fixed depth; pruning at the wrong layer discards tokens that would have become useful later. Directly relevant to the semantic diffusion risk flagged on 6/18 (Su et al.): if late-layer features lose spatial discriminability via semantic diffusion, surf's contrastive scorer must tap earlier layers. This paper gives a concrete mechanism for layer-adaptive selection and motivates asking: at which ViT depth should surf's selection signal be computed?

- [LightVLA: Differentiable Token Pruning for Efficient Vision-Language-Action Models](https://www.semanticscholar.org/paper/6163650edbfba0a837b0722c9c49ed49040ba70d) — Jiang, Jiang et al. · Semantic Scholar
  End-to-end differentiable token pruning trained jointly with the VLA policy — the pruning signal adapts as the policy learns. Risk for surf: if learned differentiable pruning (without geometric priors or language conditioning) already achieves strong efficiency on VLA tasks, surf must show that its structural prior and contrastive scorer improve over this baseline. Key question: does LightVLA's learned pruning implicitly rediscover geometric structure, or does it converge to attention-based scoring?

**adjacent**
- [ACTIVE-o3: Empowering MLLMs with Active Perception via Pure Reinforcement Learning](https://www.semanticscholar.org/paper/aae1df4156ae2088e9abb5905902b1da7eb7b6bf) — Zhu, Zhong et al. · Semantic Scholar
  Trains MLLMs to perform active perception (where and how to look, including zoom/pan) via pure RL without geometric priors or contrastive training. Complements SEE-SAL (RL-for-sufficiency, 6/17): ACTIVE-o3 learns the full spatial-control active-perception loop while SEE-SAL learns region sufficiency gating. Both are RL-based alternatives to surf's prior+contrastive approach. The open question: do ACTIVE-o3's RL-trained spatial decisions converge on the same regions surf's structural prior proposes, or on different ones?

- [ResPrune: Text-Conditioned Subspace Reconstruction for Visual Token Pruning in Large Vision-Language Models](https://www.semanticscholar.org/paper/d8d6855727d59b8f55efe9cade6191b073037d6a) — Li, Zheng et al. · Semantic Scholar
  Text-conditioned visual token pruning via subspace reconstruction: keeps tokens whose features best reconstruct the text-conditioned visual subspace. A direct risk for [[claims/language helps]] — text-conditioned selection without geometric priors or contrastive training already exists. Surf's differentiation must be specific: does the geometric structural prior add over text-conditioned reconstruction alone? Does step-adaptive conditioning (as motivated by When Does Language Matter?, 6/18) provide signal that static text conditioning cannot?

- [DepthCache: Depth-Guided Training-Free Visual Token Merging for Vision-Language-Action Model Inference](https://www.semanticscholar.org/paper/349a9a94f5d47624a65a84e9125fd430f18f6fea) — Li, Ma et al. · Semantic Scholar
  Uses depth maps as a structural prior for training-free token merging in VLAs — merges tokens with similar depth profiles, preserving spatial reasoning essential for contact-rich manipulation. Concrete comparison point for surf's prior module: depth vs. line-drawing as geometric prior, training-free merging vs. end-to-end contrastive alignment. If DepthCache's depth prior is already sufficient for spatial reasoning, what does surf's line-drawing prior add (beyond availability without depth sensors)?

**horizon**
- [HumanScale: Egocentric Human Video Can Outperform Real-Robot Data for Embodied Pretraining](https://arxiv.org/abs/2606.20521v1) — Ma, Bi et al. · arXiv

Horizon relevance for surf: egocentric human video implicitly contains gaze information (the camera points where the person looks). A data-efficiency angle not yet considered in [[eval/gaze]] or the reading list.

## questions

- GSPA geometric prior type: what is GSPA's "geometric prior" — line drawing, depth map, 3D backbone feature, or 2D edge signal? Does it operate at patch level or scene level? If patch-level geometric evidence + semantic attention reweighting, this overlaps precisely with surf's combined claim. Answer required from methods section before writing spec.
- LightVLA learned pruning convergence: does LightVLA's differentiable pruning signal correlate with geometric structure (edges, object boundaries) as training progresses, or does it converge to attention-activation scores? If it implicitly learns geometric priors, surf's explicit prior may be scaffolding for what a learned system finds anyway — weakening the novelty claim unless the prior provides sample efficiency or generalization gains.
- ACTIVE-o3 action space: is ACTIVE-o3's RL active perception formulated as spatial crop selection (zoom/pan over image) or as patch-level token selection? If spatial crop, it composes with surf's within-crop patch selection. If patch-level, it's a direct RL baseline for surf's contrastive scorer.
- ResPrune step-specificity: does ResPrune's text-conditioned reconstruction use a fixed instruction embedding or update conditioning at each VLA step? If fixed, it cannot handle step-specific language sensitivity (When Does Language Matter?, 6/18) — which surf's step-adaptive scorer would address.
- DepthCache depth availability: does DepthCache require an active depth sensor or does it use monocular depth estimation? If monocular, the structural information it uses is obtainable from RGB, narrowing the practical gap between depth-guided and line-guided priors.
