# Claude Code

Read `AGENTS.md` first.

If `$VAULT_DIR` is set (cluster context), also read `CLUSTER.md`.

Use `.agents/<skill>/SKILL.md` for named workflows.

Default to propose-only before editing vault files.

Preferred Claude use:
- skeptical review
- literature synthesis
- note refactor
- novelty / rejection-risk critique
- long-context project critique
- writing critique

Rules:
- Do not mutate files unless explicitly asked.
- Do not create, move, rename, delete, or rewrite notes without permission.
- Preserve Obsidian links.
- Keep filenames short.
- Do not invent citations or paper claims.
- Separate evidence, inference, and speculation.
- Use `git diff` / `git status` before summarizing applied changes when files are edited.
